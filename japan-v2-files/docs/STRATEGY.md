## 基本理念

**情報は不確実性を減少させる。**

投資で勝つために:

1. **情報精度の最大化** — 噂と事実を区別し、一次情報源で検証する
2. **情報の非対称性を活用** — 他者が持たない知識で優位性を得る
3. **情報の鮮度を管理** — 古いデータに基づく意思決定を排除する

### 投資哲学の核心

```
正しい順序:
1. 今買うべきか？（市場リスク）
2. 何を買うか？（企業評価）
3. いくら買うか？（ポジションサイズ）

間違った順序:
1. この企業が好き → 2. 市場環境を無視 → 3. 全力投資
```

## ミッション定義

### 目的

**外国人投資家との情報格差を逆手に取り**、グローバル市場が織り込んでいない日本企業の構造的優位を特定する。

### 情報格差の3つの源泉

| # | 源泉 | 説明 | 発見方法 |
|---|------|------|----------|
| 1 | **言語の壁** | 決算説明会Q&A、業界紙、行政文書の多くが日本語のみ | 英語メディアに出ていない情報を探す |
| 2 | **サプライチェーン深層の不可視性** | Tier2/Tier3の独占は最終製品から見えにくい | 部品・素材レベルまで掘り下げる |
| 3 | **認証・品質の参入障壁** | 防衛・宇宙・原子力での長期認定プロセスは外から見えない | 認証取得企業リスト、調達実績を調査 |

### 視点

**「グローバル・ボトルネック支配」** — 世界の計算・電力・素材・精密部品・製造装置の制約点を握る日本企業が、産業収斂において不可欠なレバレッジを持つ。

### 評価の核心的問い

```
間違い: 「この企業はどのドメインに関与しているか？」

正しい問い:
- 「この技術がなくなったら何が止まるか？」
- 「これなしでは機能できない企業/産業はいくつあるか？」
- 「競合が追いつくまで何年かかるか？」
```

## 6つのドメイン

### ドメイン定義

| # | ドメイン | 主要領域 | 日本企業の強み |
|---|---------|---------|---------------|
| 1 | **エネルギー** | 原子力、送配電、蓄電、水素、変圧器 | 原子力部品、変圧器、パワー半導体 |
| 2 | **防衛** | ミサイル、航空機、艦船、弾薬、防衛エレクトロニクス | 防衛装備品、素材、センサー |
| 3 | **AI** | コンピュートインフラ、製造装置、先端材料、データセンター | 半導体製造装置、HBM材料、冷却 |
| 4 | **ロボティクス** | 産業用、サービス、ドローン、UUV | 産業用ロボット、減速機、サーボ |
| 5 | **宇宙** | ロケット、衛星、月面、デブリ除去 | ロケット部品、衛星コンポーネント |
| 6 | **サイバー** | 暗号、量子、電子戦、ISR | 量子部品、光通信、センサー |

### 収束構造

```
AI → 電力需要 → エネルギー
防衛 → 宇宙ISR → サイバー
全ドメイン → 半導体が必要 → AI/エネルギー
```

### 収束マトリックス

|          | エネルギー | 防衛 | AI | ロボティクス | 宇宙 | サイバー |
|----------|:--------:|:----:|:--:|:----------:|:----:|:------:|
| エネルギー | - | 基地電力 | DC電力 | 製造電力 | 推進 | グリッド防御 |
| 防衛 | 基地電力 | - | 自律兵器 | ドローン | ISR | 電子戦 |
| AI | DC電力 | 自律兵器 | - | 制御 | 衛星AI | 脅威検知 |
| ロボティクス | 製造電力 | ドローン | 制御 | - | 宇宙ロボ | 物理侵入 |
| 宇宙 | 推進 | ISR | 衛星AI | 宇宙ロボ | - | 衛星通信 |
| サイバー | グリッド防御 | 電子戦 | 脅威検知 | 物理侵入 | 衛星通信 | - |

### 核心テーゼ

**6ドメイン全てへの投資が同時に加速している。複数ドメインにまたがるボトルネック技術を支配する日本企業は、複合的な需要増加を経験する。日本企業は「最終製品」より「収斂の結節点となる部品・装置・素材」で強い。**

## ドメイン管理

### 新ドメイン昇格基準

「見落としスキャン」で発見された新興領域は、以下の**全て**を満たす場合、正式ドメインに昇格する:

| 基準 | 閾値 | 検証方法 |
|------|------|---------|
| 市場規模 | TAM 50兆円以上 または 年成長率30%以上 | 業界レポート、政府予測 |
| 収束性 | 既存ドメインとの収束ポイントが2つ以上 | 収束マトリックス分析 |
| 投資可能性 | 3四半期連続で候補企業が5社以上発見 | 企業スキャン結果 |
| ボトルネック存在 | 独立したボトルネック企業が存在 | 独占力分析 |

### ドメイン昇格プロセス

```
1. 見落としスキャンで候補発見
   ↓
2. 昇格基準の評価（四半期ごと）
   ↓
3. 基準充足 → 候補ドメインとして3四半期追跡
   ↓
4. 3四半期連続で基準維持 → 正式ドメインに昇格
   ↓
5. 収束マトリックス更新、評価フレームワーク調整
```

### ドメイン統合・降格基準

以下の**いずれか**を満たす場合、統合または降格を検討:

| 基準 | 閾値 | アクション |
|------|------|-----------|
| ボトルネック消失 | 独立したボトルネック企業が存在しない | 他ドメインに統合 |
| 活動停滞 | 2四半期連続で重要イベントなし | 降格検討 |
| 収束性低下 | 収束ポイントが1つ以下に減少 | 降格検討 |
| 投資不能 | 候補企業が2社未満に減少 | 降格または統合 |

## 意思決定マトリックス

### 役割と責任

| 決定事項 | 自動化（AI） | 人間レビュー | 最終承認権 |
|---------|:----------:|:----------:|:--------:|
| 候補企業の発掘 | ○ | - | - |
| 情報収集・統合 | ○ | - | - |
| 矛盾検出・アラート | ○ | - | - |
| 5軸スコアリング（初期） | ○ | ○ | - |
| 市場リスク判定 | ○ | ○ | ○ |
| Top 30選定 | - | ○ | ○ |
| ポジションサイズ決定 | - | - | **人間** |
| 売買タイミング | - | - | **人間** |
| ドメイン昇格/降格 | - | ○ | **人間** |
| フレームワーク変更 | - | - | **人間** |

### エスカレーションルール

| 状況 | エスカレーション先 | 期限 |
|------|-----------------|------|
| 市場リスクスコア 75+ | 投資責任者 | 即時 |
| Top 30企業の重大ネガティブイベント | 投資責任者 | 24時間以内 |
| 情報矛盾が3日以上未解消 | アナリスト | 即時 |
| ドメイン昇格基準充足 | 戦略チーム | 四半期レビュー |
| フレームワーク改善提案 | 戦略チーム | 月次レビュー |

### 自動化の境界

```
┌─────────────────────────────────────────────────────────────┐
│                    自動化すべきもの                         │
├─────────────────────────────────────────────────────────────┤
│ - 情報収集（Web検索、LLM呼び出し）                         │
│ - データ統合・重複排除                                      │
│ - 矛盾検出・陳腐化検出                                      │
│ - スコアリング計算                                          │
│ - レポート生成                                              │
│ - アラート発行                                              │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                 人間が判断すべきもの                        │
├─────────────────────────────────────────────────────────────┤
│ - 最終的な投資判断（買う/売る/保有）                       │
│ - ポジションサイズ                                          │
│ - 例外的状況の評価（AIが想定しないシナリオ）               │
│ - フレームワーク自体の妥当性評価                           │
│ - ドメイン定義の変更                                        │
│ - 評価軸の重み付け変更                                      │
└─────────────────────────────────────────────────────────────┘
```
