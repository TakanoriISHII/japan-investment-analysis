## 基本理念

**情報は不確実性を減少させる。**

1. **情報精度の最大化** — 噂と事実を区別し、一次情報源で検証する
2. **情報の非対称性を活用** — 他者が持たない知識で優位性を得る
3. **情報の鮮度を管理** — 古いデータに基づく意思決定を排除する

→ 詳細: [STRATEGY.md](./docs/STRATEGY.md)

## ミッション

**外国人投資家との情報格差を逆手に取り**、グローバル市場が織り込んでいない日本企業の構造的優位を特定する。

### 情報格差の3つの源泉

| # | 源泉 | 説明 | 発見方法 |
|---|------|------|----------|
| 1 | **言語の壁** | 決算説明会Q&A、業界紙、行政文書の多くが日本語のみ | 英語メディアに出ていない情報を探す |
| 2 | **サプライチェーン深層の不可視性** | Tier2/Tier3の独占は最終製品から見えにくい | 部品・素材レベルまで掘り下げる |
| 3 | **認証・品質の参入障壁** | 防衛・宇宙・原子力での長期認定プロセスは外から見えない | 認証取得企業リスト、調達実績を調査 |

### 6つのドメイン

| # | ドメイン | 主要領域 |
|---|---------|---------|
| 1 | **エネルギー** | 原子力、送配電、蓄電、水素、変圧器 |
| 2 | **防衛** | ミサイル、航空機、艦船、弾薬、防衛エレクトロニクス |
| 3 | **AI** | コンピュートインフラ、製造装置、先端材料、データセンター |
| 4 | **ロボティクス** | 産業用、サービス、ドローン、UUV |
| 5 | **宇宙** | ロケット、衛星、月面、デブリ除去 |
| 6 | **サイバー** | 暗号、量子、電子戦、ISR |

→ 詳細: [STRATEGY.md - 6つのドメイン](./docs/STRATEGY.md#6つのドメイン)

## 評価フレームワーク概要

```
1. 市場リスク分析（100点） → 今は投資すべきタイミングか？
2. ボトルネック独占評価（5軸/100点） → どの企業が最も優位か？
3. 4象限配置 → 確実性 × アップサイドでポジション決定
```

### ボトルネック独占評価（5軸）

| 軸 | 配点 | 核心的問い |
|----|------|------------|
| **技術ボトルネック度** | 40 | この技術がないと何が止まるか？何社が依存？ |
| **競争優位持続性** | 25 | 外国企業が追いつくのに何年かかるか？ |
| **収斂加速性** | 20 | 6ドメイン全ての成長がこの技術需要を押し上げるか？ |
| **資本効率** | 10 | ROE、FCF（事実ベースのみ） |
| **非対称性** | 5 | このボトルネックは市場に認知されているか？ |

## 実行プロトコル概要

### フル分析の9フェーズ

| Phase | 内容 | 出力 |
|-------|------|------|
| 1 | 状態確認 | - |
| 2 | インテリジェンス収集（4LLM） | `data/intelligence/raw/` |
| 3 | 比較分析 | `contradictions.json`, `gaps.json` |
| 4 | 補足調査 | `claude_supplemental.md` |
| 5 | 統合・確度整理・stable更新 | `consolidated_*.json`, `stable/*` |
| 6 | 分析・評価 | `market_risk.json`, `top30.json` |
| 7 | レポート生成 | `reports/[DATE]/` |
| 8 | 継続的改善 | `reinforcement/` |
| 9 | 完了 | `state.json` 更新 |

→ 詳細: [OPERATIONS.md - フル分析プロトコル](./docs/OPERATIONS.md#フル分析プロトコル)

### フェーズ実行ルール【必須】

**各フェーズ開始時、以下の手順を必ず実行すること:**

1. **チェックリスト読み込み**: OPERATIONS.mdの該当フェーズのチェックリストを読む
2. **Todo展開**: TodoWriteで全チェック項目をTodoとして展開する
3. **逐次完了**: 各項目完了時にTodoを「completed」に更新する
4. **完了確認**: 全項目が完了するまで次フェーズに進まない

```
例: Phase 8開始時
1. Read: docs/OPERATIONS.md（Phase 8チェックリスト部分）
2. TodoWrite: [
     {"content": "8-1: Claude自動補強実行", "status": "pending"},
     {"content": "8-2: learnings/[DATE].json作成", "status": "pending"},
     {"content": "8-3: 収集プロンプト改善検討", "status": "pending"},
     ...
   ]
3. 各項目を順次実行し、完了時にTodoを更新
```

> **警告**: チェックリストをスキップすると、補強プロンプト未生成などの重要な漏れが発生する。

## 情報収集の7カテゴリ

全ての分析は**幅広い情報収集**から始まる。企業ありきではなく、情報から企業を発見する。

| # | カテゴリ | 収集内容 | 活用先 |
|---|----------|----------|--------|
| A | **国際情勢** | 地政学、政策、規制、外交、紛争 | 市場危険度、ドメイン評価 |
| B | **市場危険度シグナル** | 金融政策、レバレッジ、バリュエーション、VIX | 市場危険度分析 |
| C | **メガトレンド** | AI、脱炭素、人口動態、宇宙開発、安全保障 | ドメイン成長性、企業発掘 |
| D | **6ドメインの成長性・将来性** | 各ドメインの市場規模、成長率、投資動向 | 銘柄評価（シナジー） |
| E | **大型投資パイプライン** | 政府予算、企業CAPEX、インフラ計画 | 企業発掘、独占力評価 |
| F | **他ドメインの見逃しスキャン** | 6ドメイン外の新興領域、技術ブレイクスルー | 新規ドメイン候補 |
| G | **その他レポートに必要な情報** | 為替予測、金利見通し、センチメント | 総合判断 |

## 情報ソースの優先順位

### 一次情報（単独採用可）

| ソース | 例 | 言語 |
|--------|-----|------|
| 日本政府・行政 | 経産省、防衛省、金融庁、原子力規制委員会 | 日本語 |
| 企業開示 | EDINET、適時開示、決算短信、有価証券報告書 | 日本語 |
| 米国政府 | DOE、DoD、Commerce、SEC | 英語 |

### 言語別の情報価値

| 情報カテゴリ | 優先言語 | 非対称性 | 理由 |
|--------------|----------|----------|------|
| 日本政府政策 | 日本語 | 高 | 英訳されない詳細が多い |
| 企業決算Q&A | 日本語 | 高 | 英語transcriptがない場合が多い |
| 業界紙報道 | 日本語 | 高 | 翻訳されない |
| サプライチェーン詳細 | 日本語 | 高 | Tier2/3情報は日本語のみ |
