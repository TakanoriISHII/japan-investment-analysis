# パフォーマンス検証システム

## 概要

投資分析フレームワークの予測精度を継続的に検証し、改善するためのシステム。

## ディレクトリ構造

```
performance/
├── predictions/           # 予測記録（分析時に自動生成）
│   └── [YYYY-MM-DD]/
│       ├── top30_snapshot.json    # 選定時のスコア・予測
│       ├── market_risk_snapshot.json  # 市場リスク予測
│       └── catalysts.json         # 予測カタリスト
├── actuals/               # 実績記録（検証時に記録）
│   └── [YYYY-MM-DD]/
│       ├── price_performance.json # 株価実績
│       ├── catalyst_outcomes.json # カタリスト結果
│       └── events_occurred.json   # 発生イベント
└── analysis/              # 検証分析
    ├── monthly_review.json
    ├── quarterly_review.json
    └── annual_review.json
```

## 検証スケジュール

| 期間 | 検証対象 | 成功基準 |
|------|---------|---------|
| 1ヶ月後 | カタリスト実現、短期株価変動 | 予測イベントの50%以上が発生 |
| 3ヶ月後 | 非対称性解消、認識トリガー | 高非対称性銘柄の20%以上で解消 |
| 6ヶ月後 | 確実性評価の的中率 | 高確実性銘柄の70%以上で想定通り |
| 1年後 | ドメインテーゼの妥当性 | Top 30がベンチマークをアウトパフォーム |

## 使用方法

### 1. 予測記録（Phase 9で自動実行）

```
run analysis → Phase 9で自動的に predictions/[DATE]/ に保存
```

### 2. 実績記録（検証時に手動）

```bash
# 1ヶ月後の検証
record actuals --date 2026-01-29 --period 1m

# 3ヶ月後の検証
record actuals --date 2026-01-29 --period 3m
```

### 3. 検証レポート生成

```bash
# 月次検証
verify performance --date 2026-01-29 --period 1m

# 四半期検証
verify performance --date 2026-01-29 --period 3m
```

## 検証項目

### 軸別予測力分析

| 軸 | 検証方法 | 調整トリガー |
|----|---------|-------------|
| 構造的優位性 | スコア上位 vs 下位のリターン比較 | 相関 < 0.3 なら重み再検討 |
| 本質的価値 | バリュエーション適正度とリターンの相関 | 割安判定の精度 |
| 将来の価値 | ドメイン成長との相関 | ドメイン成長と連動しない場合 |
| 情報非対称性 | 認識トリガー発動後のリターン | 認識後も上昇しない場合 |
| 政策・触媒 | 政策変更・カタリスト発動時のパフォーマンス | 触媒が機能しない場合 |
