# CLAUDE.md - マクロパルス × マルチドメイン・コンバージェンス分析

## プロジェクト概要

このリポジトリは、日本企業のグローバル競争優位を分析し、投資判断に資するレポートを生成するシステムです。
5つのPhaseに分割された分析を順次実行し、結果をGitHubにコミットします。

---

## コマンド一覧

### フル分析（推奨）
```
フル分析を実行
```
→ Phase 1〜5を順次実行し、全結果をコミット・プッシュ

### 部分実行
```
Phase 1のみ実行（市場危険度）
Phase 2のみ実行（独占マップ）
Phase 3から再実行
Phase 5のみ実行（レポート統合）
```

### 更新系
```
週次更新（市場リスクのみ）
月次更新（Phase 3-5）
四半期更新（Phase 2-5）
```

### ユーティリティ
```
最新レポートを表示
前回との差分を表示
```

## 実行プロトコル

### 「フル分析を実行」と言われた場合

以下の手順を**自動的に**実行せよ：

```
1. 準備
   - 今日の日付でreports/YYYY-MM-DD/フォルダを作成
   - 作業開始をログ

2. Phase 1: 市場危険度スコア
   - prompts/phase1_market_risk.mdを読み込み
   - 指定の検索を実行
   - reports/YYYY-MM-DD/01_market_risk.mdに出力
   - git add & commit "Phase 1: 市場危険度スコア完了"

3. Phase 2: グローバル独占マップ
   - prompts/phase2_monopoly_map.mdを読み込み
   - 指定の検索を実行
   - reports/YYYY-MM-DD/02_monopoly_map.mdに出力
   - git add & commit "Phase 2: グローバル独占マップ完了"

4. Phase 3: マクロパルス情報収集
   - prompts/phase3_macro_pulse.mdを読み込み
   - 02_monopoly_map.mdの投資候補リストを参照
   - 指定の検索を実行
   - reports/YYYY-MM-DD/03_macro_pulse.mdに出力
   - git add & commit "Phase 3: マクロパルス情報収集完了"

5. Phase 4: Top30スコアリング
   - prompts/phase4_top30_scoring.mdを読み込み
   - 02_monopoly_map.md, 03_macro_pulse.mdを参照
   - スコアリングを実行（検索は最小限）
   - reports/YYYY-MM-DD/04_top30_ranking.mdに出力
   - git add & commit "Phase 4: Top30スコアリング完了"

6. Phase 5: 最終レポート統合
   - prompts/phase5_final_report.mdを読み込み
   - 01〜04の全ファイルを参照
   - 追加検索を実行
   - reports/YYYY-MM-DD/05_final_report.mdに出力
   - git add & commit "Phase 5: 最終レポート完了"

7. 完了処理
   - latest/フォルダを今回のレポートで更新
   - git add & commit "レポート完了: YYYY-MM-DD"
   - git push origin main
   - 完了サマリーを表示
```

### 各Phase実行時の注意事項

**検索の実行：**
- 各Phaseのプロンプトに記載された検索クエリを順次実行
- 検索結果はファイルに直接書き込まず、分析結果のみを記載
- エラー時は3回までリトライ、それでも失敗なら「検索失敗」と明記して続行

**ファイル参照：**
- 前Phaseの出力ファイルは必ず読み込んでから分析開始
- 参照したファイルのパスと作成日時を出力ファイルに記載

**Git操作：**
- 各Phase完了時に必ずcommit（pushは最後にまとめて）
- コミットメッセージは日本語で簡潔に

## エラーハンドリング

### 検索失敗時
1. 3秒待機後にリトライ
2. 3回失敗したら「検索失敗: [クエリ]」と明記して続行
3. 重要な情報が欠落した場合は、Phase完了時に警告を出力

### ファイル参照失敗時
1. 前Phaseが未実行の可能性 → 「Phase Xを先に実行してください」と通知
2. ファイルパスの確認 → 正しいパスで再試行

### Git操作失敗時
1. リモートとの同期確認 → `git pull --rebase` を試行
2. コンフリクト発生時 → ユーザーに通知して手動解決を依頼
