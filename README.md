## 1. このシステムは何をするのか

### 解決する問題

外国人投資家は日本企業の**真の競争優位**を見落としている。理由は3つ：

| 情報格差の源泉 | 説明 | 例 |
|----------------|------|-----|
| **言語の壁** | 決算Q&A、業界紙、行政文書が日本語のみ | 経産省の政策文書、電波新聞 |
| **サプライチェーン深層** | Tier2/3の独占は最終製品から見えない | ABF（味の素）、減速機（ナブテスコ） |
| **認証・参入障壁** | 防衛・宇宙・原子力の長期認定プロセス | 防衛認定に10年、原子力認証 |

### システムの目的

この情報格差を**逆手に取り**、グローバル市場が織り込んでいない日本企業の構造的優位を特定する。

### 出力物

| ファイル | 内容 |
|----------|------|
| `latest/final_report.md` | 最終レポート（Top30、市場分析） |
| `data/analysis/top30.json` | Top30ランキング（4軸スコア付き） |
| `data/analysis/market_risk.json` | 市場危険度スコア（100点満点） |

## 2. 基本思想：情報戦としての投資

### 根本原則

```
投資 = 情報戦

1. 外国人投資家が見えていない情報を見つける
2. 確度の低い情報に基づく判断を避ける
3. 歴史的パターンを学び、同じ過ちを繰り返さない
```

### 「何を買うか」の前に「今買うべきか」

**市場危険度分析（Step 4）→ 銘柄選定（Step 5）**の順序は絶対。

バフェットの教え：「価値ある株が見つからないときはキャッシュポジションのまま様子をみる」

市場危険度が高い時に優良株を買っても、全体の下落に巻き込まれる。

### バフェット的企業評価基準

| 基準 | 具体値 | 根拠 |
|------|--------|------|
| ROE | **15%以上** | バフェットが重視する最低ライン |
| 負債・資本比率 | **33%以下** | 金利上昇耐性 |
| FCF | **継続的にプラス** | オーナー収益の創出 |

加えて「経営陣の質」「競争優位の持続可能性」「顧客ロイヤルティ」を評価。

## 3. 情報の4次元評価フレームワーク

全ての情報を以下の4軸で評価する。**確度の低い情報に基づく投資判断は、判断しないより悪い。**

### 3.1 確度（Certainty）— その情報はどの段階か

| ステージ | スコア | 定義 | 例 |
|----------|--------|------|-----|
| `rumor` | 10-20 | 噂、観測記事、関係者談 | 「○○を検討か」 |
| `report_unconfirmed` | 30-40 | 報道あるが一次資料未確認 | 「○○する方針」 |
| `report_confirmed` | 50-60 | 複数報道で確認、一次資料なし | 「○○で調整」 |
| `official_announcement` | 70-80 | 公式発表あり | 「○○を発表」 |
| `contract_signed` | 85-95 | 契約締結、予算成立、法案成立 | 「○○で合意」 |
| `executed` | 95-99 | 実行済み、納品済み、決算で確定 | 「○○が完了」 |

### 3.2 検証度（Verification）— どこまで裏取りしたか

| レベル | 定義 | 信頼度 | 単独採用 |
|--------|------|--------|----------|
| `primary_confirmed` | 一次資料（政府発表、企業開示）で確認 | 最高 | ✓ 可 |
| `multi_source` | 独立した2ソース以上で確認 | 高 | ✓ 可 |
| `single_reliable` | 単一だが信頼できるソース | 中 | △ 条件付き |
| `single_unverified` | 単一ソース、未検証 | 低 | ✗ 不可 |
| `conflicting` | 複数ソースで情報が矛盾 | 要注意 | ✗ 不可 |

**単独採用禁止**: SNS、掲示板、個人ブログ、アナリスト株価予想

### 3.3 非対称性（Asymmetry）— 誰が知っていて、誰が知らないか

| スコア | 状態 | 説明 |
|--------|------|------|
| 21-25 | 極めて高い | 3源泉全てで非対称性あり、ほぼ未認知 |
| 16-20 | 高い | 2源泉で非対称性、一部専門家のみ認知 |
| 11-15 | 中程度 | 1源泉で非対称性、日本市場専門家は認知 |
| 6-10 | 低い | 一部グローバル投資家も認知 |
| 0-5 | なし | 広く認知済み、織り込み済み |

**非対称性の解消リスク**: 英語メディア報道、アナリストレポート発行で解消。**解消前に投資判断を行う必要がある**。

### 3.4 鮮度（Freshness）— いつの情報か、いつまで有効か

| 情報タイプ | 減衰速度 | 有効期間目安 |
|------------|----------|--------------|
| 株価・為替 | 極めて速い | 当日のみ |
| 需給（外国人売買等） | 速い | 1週間 |
| 政策発表 | 中程度 | 実行まで |
| 決算・ガイダンス | 中程度 | 次回決算まで |
| 世界シェア | 遅い | 6ヶ月〜1年 |
| 認証・参入障壁 | 極めて遅い | 数年 |

## 4. 銘柄評価：4軸スコアリング

Top30の選出は以下の4軸で評価（**100点満点**）：

| 軸 | 配点 | 説明 | 評価ポイント |
|----|------|------|--------------|
| **独占力** | 30点 | グローバル独占力 | シェア、参入障壁、代替技術の有無 |
| **資本効率** | 20点 | 財務健全性 | ROE、負債比率、FCF |
| **シナジー** | 25点 | 領域横断効果 | 6ドメインの収斂から恩恵を受けるか |
| **非対称性** | 25点 | 情報格差 | 言語、SC深層、認証の3源泉 |

### 4象限マトリックス

銘柄を「確度」と「非対称性」の2軸で分類：

```
              非対称性 高(16+)
                  ↑
    ┌─────────────┼─────────────┐
    │ 【要検証】    │ 【最優先】    │
    │ 確度<70     │ 確度≥70     │
    │ 非対称性≥16  │ 非対称性≥16  │
    │             │             │
    │ 追加調査で   │ 即座に       │
    │ 確度を上げる │ ポジション構築 │
確度├─────────────┼─────────────┤確度
<70 │ 【様子見】   │ 【安定】     │ ≥70
    │ 確度<70     │ 確度≥70     │
    │ 非対称性<16  │ 非対称性<16  │
    │             │             │
    │ 情報不足    │ αは小さいが  │
    │ 判断保留    │ 確実性高い   │
    └─────────────┴─────────────┘
              非対称性 低(<16)
```

| 象限 | アクション |
|------|------------|
| **最優先** | 確度も非対称性も高い。即座にポジション構築を検討 |
| **安定** | 確度は高いが非対称性は低い。αは小さいが確実性高い |
| **要検証** | 非対称性は高いが確度が低い。追加調査で確度を上げる |
| **様子見** | 両方低い。情報不足で判断保留 |

## 5. 市場危険度分析

### 目的

「今買うべきか」を判断する。100点満点でスコアリング。

### Tier別重みづけ（歴史的破壊力に基づく）

| Tier | 項目 | 配点 | 根拠 |
|------|------|------|------|
| **Tier1** | 金融政策転換リスク | 15点 | 1990年バブル崩壊、2024年8月暴落 |
| **Tier1** | レバレッジ崩壊リスク | 15点 | 円キャリー巻き戻し |
| **Tier2** | バリュエーション過熱 | 10点 | PER/PBR異常高騰 |
| **Tier2** | VIX・ボラティリティ | 10点 | 低位安定は逆に警戒 |
| **Tier2** | 外国人売買動向 | 10点 | 需給の先行指標 |
| **Tier3** | 中国・新興国リスク | 10点 | 中国減速、通貨危機 |
| **Tier3** | 地政学リスク | 10点 | 台湾海峡、テロ等 |
| **Tier3** | 市場流動性 | 10点 | 売りが売りを呼ぶ |
| **Tier3** | 為替リスク | 10点 | 円高/円安の影響 |

**均等配分は怠慢**。歴史的に最も破壊的だった金融政策転換とレバレッジ崩壊に高い配点。

### バフェット的投資指針

| 市場危険度 | 解釈 | 推奨 |
|------------|------|------|
| 0-25 | 積極投資の好機 | フルポジション |
| 26-45 | 優良株を選別 | 通常ポジション |
| 46-60 | やや慎重姿勢 | 60-80%投資 |
| 61-75 | キャッシュを増やす | ポジション縮小 |
| 76-100 | 様子見、現金化 | 大幅縮小 |

### 歴史的暴落パターン（毎回チェック）

| パターン | 主因 | 先行指標 |
|----------|------|----------|
| バリュエーション過熱型 | PER/PBR異常高騰 | PER30倍超 |
| 金融政策転換型 | 中央銀行タカ派転換 | 利上げシグナル |
| レバレッジ崩壊型 | キャリートレード巻戻し | ポジション膨張 |
| 外部ショック型 | テロ、災害、パンデミック | 突発的 |
| 新興国危機型 | 中国減速、通貨危機 | 新興国指標 |
| アルゴ・流動性危機型 | 流動性低下 | 出来高急減 |

## 6. 分析フロー（Step 0〜6）

### 全体フロー

```
Step 0: 企業発掘（70-100社）
    │ 認証リスト法、サプライチェーン逆引き法、ボトルネック・ニュース法
    ↓ 一次スクリーニング
候補プール（50社）
    │
    ↓
Step 1: イベント収集（4次元評価付き）
    ↓
Step 1.5: 情報品質チェック（矛盾・陳腐化検知）
    ↓
Step 2: 独占マップ更新
    ↓
Step 3: 市場状態取得
    ↓
Step 4: 市場危険度分析 ← 「今買うべきか」を判断
    ↓
Step 5: Top30スコアリング（4軸評価・100点満点）
    │
    ├→ Top30選出
    └→ 監視リスト(31-50位)
    ↓
Step 6: 最終レポート生成
```

### 各Stepの詳細

| Step | 処理内容 | 出力 |
|------|----------|------|
| **Step 0** | 70-100社を発掘、スクリーニングで50社に絞り込み | `data/discovery/` |
| **Step 1** | 候補50社のイベント収集、4次元評価を付与 | `data/events.json` |
| **Step 1.5** | 矛盾検知、陳腐化検知、確度の自動調整 | `data/information_quality.json` |
| **Step 2** | 独占マップに50社を反映 | `data/snapshots/monopoly_map.json` |
| **Step 3** | 株価、為替、指数などの動的データ取得 | `data/snapshots/market_state.json` |
| **Step 4** | Tier別重みづけで市場危険度をスコアリング | `data/analysis/market_risk.json` |
| **Step 5** | 4軸評価で50社をランキング、Top30選出 | `data/analysis/top30.json` |
| **Step 6** | 最終レポート生成 | `reports/[日付]/`, `latest/` |

### 6ドメインのカバレッジ

| ドメイン | 含まれる領域 | 日本企業の強み |
|----------|--------------|----------------|
| **AI** | 半導体製造装置、先端材料、DC | 装置シェア、材料独占 |
| **エネルギー** | 原子力、送配電、蓄電、水素 | 原子力部品、変圧器 |
| **軍事・防衛** | ミサイル、航空機、艦船、弾薬 | 防衛装備品、認証障壁 |
| **ロボティクス** | 産業用、サービス、ドローン | 減速機、サーボモータ |
| **宇宙** | ロケット、衛星、月面 | 部品、コンポーネント |
| **サイバー** | 暗号、量子、電子戦 | 量子部品、光通信 |

## 7. 使い方

### クイックスタート

```bash
# 1. リポジトリをクローン
git clone https://github.com/TakanoriISHII/japan-investment-analysis.git
cd japan-investment-analysis

# 2. Claude Codeで開く
claude

# 3. 分析を実行
> フル分析を実行
```

### コマンド一覧

| コマンド | 動作 |
|----------|------|
| `フル分析を実行` | 状態を確認し、必要な部分のみ更新して分析 |
| `強制フル更新` | 全てをゼロから再実行（キャッシュ無視） |
| `企業発掘を実行` | Step 0のみ実行、新たな候補を発掘 |
| `情報品質を確認` | 未検証・矛盾・陳腐化アラートを表示 |
| `非対称性を確認` | アクティブな非対称性と解消リスクを表示 |
| `状態を確認` | 各コンポーネントの最終更新日時を表示 |
| `差分レポート` | 前回からの変化点のみ表示 |

### 最新結果の確認

```bash
# 最終レポート
cat latest/final_report.md

# 市場危険度
cat data/analysis/market_risk.json

# Top30ランキング
cat data/analysis/top30.json
```

## 9. 設計原則（Claude Codeへの引継ぎ）

### 絶対に変更しない原則

1. **「今買うべきか」→「何を買うか」の順序**
   - 市場危険度分析（Step 4）が先、銘柄選定（Step 5）が後

2. **重みづけの根拠を明記**
   - 歴史的破壊力に基づく配点（均等配分は怠慢）

3. **バフェット基準の具体値**
   - ROE 15%以上、負債比率33%以下

4. **情報品質フレームワーク**
   - 確度・検証度・非対称性・鮮度の4次元評価を維持

5. **非対称性の3源泉**
   - 言語、サプライチェーン深層、認証を常に意識

6. **Step間の依存関係**
   - Step 4の結果がStep 5に影響、Step 1.5は省略しない

### 将来の拡張候補

- トランプ関税など**政策リスクの体系的追跡**
- **セクターローテーション**の検知
- **個別銘柄のイベントカレンダー**
- **為替シナリオ別の影響分析**の自動化
