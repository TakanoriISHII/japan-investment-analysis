## 1. このシステムは何をするのか

### 解決する問題

外国人投資家は日本企業の**真の技術ボトルネック**を見落としている。理由は3つ：

| 情報格差の源泉 | 説明 | 例 |
|----------------|------|-----|
| **言語の壁** | 決算Q&A、業界紙、行政文書が日本語のみ | 経産省の政策文書、電波新聞 |
| **サプライチェーン深層** | Tier2/3の独占は最終製品から見えない | ABF、減速機、ガスケット |
| **認証・参入障壁** | 防衛・宇宙・原子力の長期認定プロセス | 防衛認定に10年、原子力認証 |

### システムの目的

この情報格差を**逆手に取り**、グローバル市場が織り込んでいない日本企業の「技術ボトルネック独占」を特定する。

### 評価の核心

```
❌ 「この企業は何ドメインに関与しているか？」
⭕ 「この技術がなくなったら、何が止まるか？」
⭕ 「何社・何業界がこの技術なしでは成り立たないか？」
```

**NVIDIA例**: ゲーム・AI・DC・自動運転全てがこのGPUを「必要とする」
**Starlink例**: 通信・軍事・EV・ロボット全てがこのインフラを「必要とする」可能性

### 出力物

| ファイル | 内容 |
|----------|------|
| `latest/final_report.md` | 最終レポート（Top30、市場分析） |
| `latest/executive_summary.md` | エグゼクティブサマリー（1ページ要約） |
| `latest/quadrant_chart.html` | **4象限マトリックス可視化（インタラクティブ）** |
| `data/analysis/top30.json` | Top30ランキング（ボトルネック評価スコア付き） |
| `data/analysis/market_risk.json` | 市場危険度スコア（100点満点） |

## 2. 基本思想：情報起点のアプローチ

### 根本原則

```
投資 = 情報戦

情報は不確実性を減少させるものである。
1. 情報の確度を極限まで高める — 噂と事実を峻別し、一次資料で検証
2. 情報の非対称性を突く — 他者が知らない情報で優位に立つ
3. 情報の鮮度を管理する — 古い情報に基づく判断を排除する
```

### 情報起点 vs 企業起点

```
✓ 情報起点（このシステム）:
  幅広い情報収集 → 情報から企業発見 → 企業深掘り → 評価

✗ 企業起点（避けるべき）:
  企業リスト → その企業の情報収集 → 評価
```

情報が先、企業が後。**情報から企業を発見する**アプローチで、既知のバイアスを排除する。

### マルチLLMアーキテクチャ

複数のLLMから情報を収集し、Claudeが統合・分析する:

```
┌─────────────────────────────────────────────────────────┐
│  外部LLM（情報収集係）                                   │
│  - Claude, Gemini, GPT-4, Grok 等                       │
│  - 2種類の情報を収集:                                    │
│    1. company_info/: 企業・独占・財務情報               │
│    2. broad_info/: 広域情報（市場・政策）               │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Claude（分析係）                                        │
│  - 全LLMの出力を読み込み・統合                           │
│  - 重複排除、矛盾解決、4次元評価                         │
│  - ★ボトルネック評価、Top30選出、レポート生成           │
│  - HTML可視化（4象限マトリックス）                       │
└─────────────────────────────────────────────────────────┘
```

**データ保存先**:
- `data/intelligence/raw/company_info/[llm名]_cd.md`
- `data/intelligence/raw/broad_info/[llm名]_mi.md`

### 「何を買うか」の前に「今買うべきか」

**市場危険度分析 → 銘柄選定**の順序は絶対。

バフェットの教え：「価値ある株が見つからないときはキャッシュポジションのまま様子をみる」

市場危険度が高い時に優良株を買っても、全体の下落に巻き込まれる。

## 3. 情報収集の7カテゴリ

全ての分析は**幅広い情報収集**から始まる。

| # | カテゴリ | 収集内容 | 活用先 |
|---|----------|----------|--------|
| A | **国際情勢** | 地政学、政策、規制、外交、紛争 | 市場危険度、ドメイン評価 |
| B | **市場危険度シグナル** | 金融政策、レバレッジ、バリュエーション、VIX | 市場危険度分析 |
| C | **メガトレンド** | AI、脱炭素、人口動態、宇宙開発、安全保障 | ドメイン成長性、企業発掘 |
| D | **6ドメインの成長性** | 各ドメインの市場規模、成長率、投資動向 | 銘柄評価（収斂加速性） |
| E | **大型投資パイプライン** | 政府予算、企業CAPEX、インフラ計画 | 企業発掘、ボトルネック評価 |
| F | **見逃しスキャン** | 6ドメイン外の新興領域、技術ブレイクスルー | 新規ドメイン候補 |
| G | **その他** | 為替予測、金利見通し、センチメント | 総合判断 |

**全ての収集情報に4次元評価（確度・検証度・非対称性・鮮度）を付与する。**

## 4. 情報の4次元評価フレームワーク

全ての情報を以下の4軸で評価する。**確度の低い情報に基づく投資判断は、判断しないより悪い。**

### 4.1 確度（Certainty）— その情報はどの段階か

| ステージ | スコア | 定義 | 例 |
|----------|--------|------|-----|
| `rumor` | 10-20 | 噂、観測記事、関係者談 | 「○○を検討か」 |
| `report_unconfirmed` | 30-40 | 報道あるが一次資料未確認 | 「○○する方針」 |
| `report_confirmed` | 50-60 | 複数報道で確認、一次資料なし | 「○○で調整」 |
| `official_announcement` | 70-80 | 公式発表あり | 「○○を発表」 |
| `contract_signed` | 85-95 | 契約締結、予算成立、法案成立 | 「○○で合意」 |
| `executed` | 95-99 | 実行済み、納品済み、決算で確定 | 「○○が完了」 |

### 4.2 検証度（Verification）— どこまで裏取りしたか

| レベル | 定義 | 単独採用 |
|--------|------|----------|
| `primary_confirmed` | 一次資料で確認 | ✓ 可 |
| `multi_source` | 独立した2ソース以上 | ✓ 可 |
| `single_reliable` | 単一の信頼ソース | △ 条件付き |
| `single_unverified` | 単一・未検証 | ✗ 不可 |
| `conflicting` | ソース間で矛盾 | ✗ 不可 |

### 4.3 非対称性（Asymmetry）— 誰が知っているか

| スコア | 状態 | 説明 |
|--------|------|------|
| 21-25 | 極めて高い | 3源泉全てで非対称性あり |
| 16-20 | 高い | 2源泉で非対称性 |
| 11-15 | 中程度 | 1源泉で非対称性 |
| 6-10 | 低い | 一部グローバル投資家も認知 |
| 0-5 | なし | 広く認知済み |

### 4.4 鮮度（Freshness）— いつまで有効か

| 情報タイプ | 有効期間目安 |
|------------|--------------|
| 株価・為替 | 当日のみ |
| 需給 | 1週間 |
| 政策発表 | 実行まで |
| 世界シェア | 6ヶ月〜1年 |
| 認証・参入障壁 | 数年 |

## 5. 銘柄評価：ボトルネック独占評価

Top30の選出は以下の5軸で評価（**100点満点**）：

### 概要

| 軸 | 配点 | 核心の問い |
|----|------|------------|
| **技術ボトルネック度** | 40点 | この技術がないと何が止まるか？何社が依存？ |
| **競争優位持続性** | 25点 | 外国企業が追いつくのに何年かかるか？ |
| **収斂加速性** | 20点 | 6ドメイン全ての成長がこの技術需要を押し上げるか？ |
| **資本効率** | 10点 | ROE、FCF（事実ベースのみ） |
| **非対称性** | 5点 | このボトルネックは市場に認知されているか？ |

---

### 5.1 技術ボトルネック度（40点）★最重要★

**核心質問: 「この技術を使わずに〇〇できるか？」**

| 評価項目 | 配点 | 内容 |
|----------|------|------|
| **依存企業・業界数** | 15点 | 何社・何業界がこの技術なしでは製品を作れないか |
| **代替不可能性** | 15点 | 他の技術で代替できるか？ できない=満点 |
| **市場シェア** | 10点 | 独占〜寡占の度合い |

**評価時の質問:**
- この技術がなくなったら、どの企業の製品が作れなくなるか？
- この技術の顧客リストに、世界的大企業が何社含まれるか？
- 競合他社も含め、業界全体がこの技術に依存しているか？

---

### 5.2 競争優位持続性（25点）

**核心質問: 「海外企業が同等品を作るのに何年かかるか？」**

| 評価項目 | 配点 | 内容 |
|----------|------|------|
| **技術リードタイム** | 10点 | 競合が追いつくまでの年数 |
| **地理的・制度的独占性** | 8点 | 日本/特定国でしか作れない理由 |
| **スイッチングコスト** | 7点 | 顧客が他社に切り替える困難さ |

**外国競合リスクチェック（減点要素）:**
- 中国企業が同等品を開発・量産開始 → -3点
- 韓国企業が技術的にキャッチアップ中 → -2点
- 欧米企業がM&A・大規模投資で参入 → -2点

---

### 5.3 収斂加速性（20点）

**核心質問: 「6ドメイン全ての成長がこの技術需要を押し上げるか？」**

| 評価項目 | 配点 | 内容 |
|----------|------|------|
| **ドメイン横断需要** | 12点 | 各ドメインの投資増→この技術需要増？ |
| **相乗効果・価値増幅** | 8点 | ドメイン間の収斂で価値が増幅するか |

**評価方法**: 各ドメインについて「〇〇投資が増えると、この技術の需要は増えるか？」を問う

```
□ AI投資が増えると需要増？        → +2点
□ エネルギー投資が増えると需要増？ → +2点
□ 防衛投資が増えると需要増？      → +2点
□ ロボティクス投資が増えると需要増？→ +2点
□ 宇宙投資が増えると需要増？      → +2点
□ サイバー投資が増えると需要増？  → +2点
```

**重要**: 「企業が関与している」ではなく「その技術への需要が増える」で判断

**6ドメイン**: AI、エネルギー、軍事・防衛、ロボティクス、宇宙、サイバー

---

### 5.4 資本効率（10点）

**事実ベースで取得可能な情報**のみで評価。推定値・アナリスト予想は使わない。

| 評価項目 | 配点 | 基準 |
|----------|------|------|
| ROE | 4点 | 15%以上で満点 |
| FCF（継続性） | 4点 | 5年連続プラスで満点 |
| 負債比率 | 2点 | 33%以下で満点 |

**データソース**: 有価証券報告書（EDINET）、決算短信、企業IR資料のみ。

---

### 5.5 非対称性（5点）

**ボトルネック独占が市場に認知されているかどうか**のみを評価。

| 評価項目 | 配点 | 内容 |
|----------|------|------|
| **認知度の低さ** | 3点 | プロ投資家・アナリストも見逃しているか |
| **認知拡大トリガー** | 2点 | 認知が広がる明確なきっかけがあるか |

**注意**: 非対称性は「タイミング」の問題であり、ボトルネック独占の本質的価値ではない。そのため配点は5点に抑制。

---

### 評価例

| 企業 | コア技術 | ボトルネック(40) | 競争優位(25) | 収斂加速(20) | 資本効率(10) | 非対称(5) | **合計** |
|------|----------|------------------|--------------|--------------|--------------|-----------|----------|
| 味の素 | ABF層間絶縁材 | 38 | 22 | 14 | 8 | 4 | **86** |
| レーザーテック | EUVマスク検査 | 40 | 23 | 10 | 9 | 2 | **84** |
| 東京エレクトロン | コータ/デベロッパー | 35 | 20 | 16 | 9 | 1 | **81** |

## 6. 市場危険度分析

### 目的

「今買うべきか」を判断する。100点満点でスコアリング。

### Tier別重みづけ（歴史的破壊力に基づく）

| Tier | 項目 | 配点 | 根拠 |
|------|------|------|------|
| **Tier1** | 金融政策転換リスク | 15点 | 1990年バブル崩壊、2024年8月暴落 |
| **Tier1** | レバレッジ崩壊リスク | 15点 | 円キャリー巻き戻し |
| **Tier2** | バリュエーション過熱 | 10点 | PER/PBR異常高騰 |
| **Tier2** | VIX・ボラティリティ | 10点 | 低位安定は逆に警戒 |
| **Tier2** | 外国人売買動向 | 10点 | 需給の先行指標 |
| **Tier3** | 中国・新興国リスク | 10点 | 中国減速、通貨危機 |
| **Tier3** | 地政学リスク | 10点 | 台湾海峡、テロ等 |
| **Tier3** | 市場流動性 | 10点 | 売りが売りを呼ぶ |
| **Tier3** | 為替リスク | 10点 | 円高/円安の影響 |

**均等配分は怠慢**。歴史的に最も破壊的だった金融政策転換とレバレッジ崩壊に高い配点。

### バフェット的投資指針

| 市場危険度 | 解釈 | 推奨 |
|------------|------|------|
| 0-25 | 積極投資の好機 | フルポジション |
| 26-45 | 優良株を選別 | 通常ポジション |
| 46-60 | やや慎重姿勢 | 60-80%投資 |
| 61-75 | キャッシュを増やす | ポジション縮小 |
| 76-100 | 様子見、現金化 | 大幅縮小 |

## 7. 分析フロー（Phase 1〜5）

### 全体フロー

```
Phase 2: 情報収集（Claude + 外部LLM）
    │ 企業発掘、広域情報、独占情報、財務情報
    ↓
Phase 3: 統合
    │ 全LLM出力をマージ、重複排除、矛盾解決
    ↓
★Phase 3.5: Gap分析 & 補足調査 ← 見落とし防止機構
    │ ボトルネック情報チェック、外国競合リスク調査
    │ 収斂加速性チェック（1ドメインのみの企業を補足調査）
    ↓
Phase 4: 分析
    │ イベント収集 → 品質チェック → 独占マップ更新
    │ 市場危険度分析 ← 「今買うべきか」を判断
    │ Top30スコアリング（ボトルネック評価）
    ↓
Phase 5: 完了処理
    │ 最終レポート生成
    │ HTML可視化生成（4象限マトリックス）
    └→ reports/[日付]/, latest/
```

### 各Phaseの詳細

| Phase | 処理内容 | 出力 |
|-------|----------|------|
| **Phase 2** | 4タイプの情報収集（企業発掘、広域、独占、財務） | `data/intelligence/raw/` |
| **Phase 3** | 全LLM出力をマージ、重複排除 | `consolidated_*.json` |
| **Phase 3.5** | Gap分析、ボトルネック情報の補足調査 | `gap_analysis_*.json` |
| **Phase 4** | 市場危険度分析、Top30スコアリング | `market_risk.json`, `top30.json` |
| **Phase 5** | 最終レポート、HTML可視化 | `final_report.md`, `quadrant_chart.html` |

### Gap分析（Phase 3.5）のチェック項目

| チェック | 内容 | アクション |
|----------|------|------------|
| **ボトルネック情報** | 依存企業数、代替技術が調査されているか | 不足なら補足調査 |
| **競争優位持続性** | 外国競合リスクが調査されているか | 不足なら補足調査 |
| **収斂加速性** | 6ドメイン全てについて需要増加が調査されているか | 1-2ドメインのみなら補足調査 |
| **ランク乖離** | 情報収集で上位だが評価で圏外 | 情報不足の可能性を検証 |

## 8. 使い方

### クイックスタート

```bash
# 1. リポジトリをクローン
git clone https://github.com/TakanoriISHII/japan-investment-analysis.git
cd japan-investment-analysis

# 2. Claude Codeで開く
claude

# 3. 分析を実行
> フル分析を実行
```

### コマンド一覧

| コマンド | 動作 |
|----------|------|
| `フル分析を実行` | 状態を確認し、必要な部分のみ更新して分析 |
| `強制フル更新` | 全てをゼロから再実行 |
| `広域情報収集` | Phase 2のみ実行（7カテゴリ） |
| `企業発掘を実行` | 企業発掘のみ実行 |
| `ボトルネック評価を実行` | ボトルネック独占評価でTop30選出 |
| `情報品質を確認` | 未検証・矛盾・陳腐化アラートを表示 |
| `状態を確認` | 各コンポーネントの最終更新日時を表示 |
| `差分レポート` | 前回からの変化点のみ表示 |

### 最新結果の確認

```bash
# 最終レポート
cat latest/final_report.md

# 市場危険度
cat data/analysis/market_risk.json

# Top30ランキング
cat data/analysis/top30.json
```

## 10. 設計原則

### 絶対に変更しない原則

1. **情報起点のアプローチ**
   - 情報収集が先、企業選定が後

2. **「今買うべきか」→「何を買うか」の順序**
   - 市場危険度分析が先、銘柄選定が後

3. **ボトルネック評価の核心**
   - 「この技術がなくなったら何が止まるか？」
   - 「何社・何業界がこの技術に依存しているか？」
   - 「外国企業が追いつくのに何年かかるか？」

4. **5軸評価の定義**
   - 技術ボトルネック度：依存企業数、代替不可能性、シェア
   - 競争優位持続性：リードタイム、地理的独占性、スイッチングコスト
   - 収斂加速性：6ドメイン全ての成長が需要を押し上げるか
   - 資本効率：事実ベースのみ（推定値禁止）
   - 非対称性：認知度の低さ（タイミング要素なので配点抑制）

5. **重みづけの根拠を明記**
   - 技術ボトルネック度に最大配点（40点）
   - 歴史的破壊力に基づく市場危険度配点

6. **Gap分析による見落とし防止**
   - Phase 3.5でボトルネック情報の不足を検出
   - 外国競合リスクを必ず調査

7. **非対称性の3源泉**
   - 言語、サプライチェーン深層、認証を常に意識
