# マクロパルス × マルチドメイン・コンバージェンス分析

日本企業のグローバル競争優位を分析し、投資判断に資するレポートを自動生成する**情報戦略システム**。

---

## 🎯 設計者（Takanori）の投資哲学と設計思想

**Claude Codeで改善を行う際は、以下の原則を必ず遵守すること。**

### 根本思想：情報戦争としての投資

このシステムは単なるスクリーニングツールではない。**情報の非対称性を武器にした戦略的優位の構築**が目的である。

```
投資 = 情報戦
・外国人投資家が見えていない情報を見つける
・確度の低い情報に基づく判断を避ける
・歴史的パターンを学び、同じ過ちを繰り返さない
```

### 最重要原則：「今買うべきか」が先

**銘柄選定（何を買うか）の前に、市場環境（今買うべきか）を判断する。**

これはバフェットの教え「価値ある株が見つからないときはキャッシュポジションのまま様子をみる」に基づく。市場危険度が高い時に優良株を買っても、全体の下落に巻き込まれる。

### 歴史から学ぶ：暴落パターンの検知

過去の暴落には共通パターンがある。これを無視した市場分析は不完全である。

| 暴落 | 主因 | 教訓 |
|------|------|------|
| バブル崩壊（1990） | 日銀引締め | **金融政策転換が最大のトリガー** |
| リーマン（2008） | レバレッジ崩壊 | **信用膨張は急激に巻き戻る** |
| 2024年8月 | 円キャリー巻戻し | **VIX低位安定は逆に危険信号** |

**市場危険度分析（Step 4）では、これらのパターンとの類似性を毎回チェックすること。**

### 重みづけの原則：均等配分は怠慢

スコアリングにおいて、全項目を均等配分することは避ける。**過去の破壊力に基づいて重みづけする**。

例：市場危険度スコア
- **Tier1**（金融政策、レバレッジ）: 各15点 → **計30点**
- **Tier2**（バリュエーション、VIX等）: **計30点**
- **Tier3**（その他）: **計40点**

金融政策転換とレバレッジ崩壊が最も破壊的だったという歴史的事実に基づく。

### バフェット的評価基準

企業評価において、以下のバフェット基準を具体的な数値で適用：

| 基準 | 具体値 | 根拠 |
|------|--------|------|
| ROE | **15%以上** | バフェットが重視する最低ライン |
| 負債・資本比率 | **33%以下** | 金利上昇耐性 |
| FCF | **継続的にプラス** | オーナー収益の創出 |

「経営陣の質」「競争優位の持続可能性」「顧客ロイヤルティ」も重視。

### 情報品質への執着

**確度の低い情報に基づく投資判断は、判断しないより悪い。**

- 一次資料（政府発表、企業開示）を最優先
- 報道ベースの情報は必ず確度を明記
- 矛盾する情報は両論併記し、判断を保留
- 「未確認」「推定」のラベルを恐れずに付与

### 非対称性の源泉を常に意識

情報格差の3つの源泉を常に意識し、分析に活用：

1. **言語の壁** — 日本語のみの決算Q&A、業界紙、行政文書
2. **サプライチェーン深層** — Tier2/3の独占は最終製品から見えない
3. **認証・参入障壁** — 防衛・宇宙・原子力の長期認定プロセス

**これらの非対称性が解消されるイベント（英語報道、アナリストカバレッジ開始）は、投資機会の消失を意味する。**

## 📋 Claude Codeへの引継ぎ事項

### 改善時の注意点

1. **歴史的暴落パターンのデータベースを維持・拡充**
   - 新たな暴落が発生した場合は追加
   - 各暴落の「先行指標」を具体的に記録

2. **重みづけの根拠を常に明記**
   - なぜその配点なのかを説明できる状態を維持
   - 均等配分に戻さない

3. **バフェット基準の具体値を変更しない**
   - ROE 15%、負債比率33%は根拠のある数値
   - 変更する場合は明確な理由を記録

4. **情報品質フレームワークを簡略化しない**
   - 確度・検証度・非対称性・鮮度の4次元評価を維持
   - 「面倒だから」という理由で削除しない

5. **市場危険度と銘柄選定の順序を逆転させない**
   - 必ず「今買うべきか」→「何を買うか」の順序

6. **Step間の依存関係を崩さない**
   - Step 4（市場危険度）の結果がStep 5（銘柄選定）に影響
   - Step 1.5（情報品質チェック）は省略しない

### 将来の拡張候補

- トランプ関税など**政策リスクの体系的追跡**
- **セクターローテーション**の検知
- **個別銘柄のイベントカレンダー**（決算、株主総会等）
- **為替シナリオ別の影響分析**の自動化

### 設計者の連絡事項

このシステムは「完璧な予測」ではなく「情報に基づく優位性の構築」を目指している。
予測が外れることは許容するが、**確度の低い情報に基づく判断**は許容しない。

## 基本哲学

**情報は不確実性を減少させるものである。**

- 情報の**確度**を極限まで高める（一次資料で検証）
- 情報の**非対称性**を突く（他者が知らない情報で優位に立つ）
- 情報の**鮮度**を管理する（古い情報に基づく判断を排除）

## 情報の4次元評価

全ての情報を以下の4軸で評価：

| 軸 | 説明 |
|-----|------|
| **確度（Certainty）** | 噂→報道→公式発表→契約→実績 |
| **検証度（Verification）** | 単独→複数ソース→一次資料確認 |
| **非対称性（Asymmetry）** | 誰が知っているか、いつ解消されるか |
| **鮮度（Freshness）** | いつの情報か、いつまで有効か |

## 情報格差の3つの源泉

| # | 源泉 | 説明 |
|---|------|------|
| 1 | **言語の壁** | 決算Q&A、業界紙、行政文書が日本語のみ |
| 2 | **サプライチェーン深層** | Tier2/3の独占は最終製品から見えない |
| 3 | **認証・品質の参入障壁** | 防衛・宇宙・原子力の長期認定プロセス |

## 分析フロー

```
Step 1: イベント収集（4次元評価付き）
    ↓
Step 1.5: 情報品質チェック（矛盾・陳腐化検知）
    ↓
Step 2: 独占マップ更新（トリガーがあれば）
    ↓
Step 3: 市場状態取得
    ↓
Step 4: 市場危険度分析
    ↓
Step 5: Top30スコアリング（確度×非対称性マトリックス）
    ↓
Step 6: 最終レポート生成（情報品質サマリー付き）
```

## コマンド一覧

| コマンド | 動作 |
|----------|------|
| `フル分析を実行` | 状態を確認し、必要な部分のみ更新 |
| `強制フル更新` | 全てをゼロから再実行 |
| `情報品質を確認` | 未検証・矛盾・陳腐化アラートを表示 |
| `非対称性を確認` | アクティブな非対称性と解消リスクを表示 |
| `差分レポート` | 前回からの変化点のみ表示 |
