## 1. このシステムは何をするのか

### 解決する問題

外国人投資家は日本企業の**真の競争優位**を見落としている。理由は3つ：

| 情報格差の源泉 | 説明 | 例 |
|----------------|------|-----|
| **言語の壁** | 決算Q&A、業界紙、行政文書が日本語のみ | 経産省の政策文書、電波新聞 |
| **サプライチェーン深層** | Tier2/3の独占は最終製品から見えない | ABF、減速機、ガスケット |
| **認証・参入障壁** | 防衛・宇宙・原子力の長期認定プロセス | 防衛認定に10年、原子力認証 |

### システムの目的

この情報格差を**逆手に取り**、グローバル市場が織り込んでいない日本企業の構造的優位を特定する。

### 出力物

| ファイル | 内容 |
|----------|------|
| `latest/final_report.md` | 最終レポート（Top30、市場分析） |
| `latest/executive_summary.md` | エグゼクティブサマリー（1ページ要約） |
| `latest/quadrant_chart.html` | **4象限マトリックス可視化（インタラクティブ）** |
| `data/analysis/top30.json` | Top30ランキング（4軸スコア付き） |
| `data/analysis/market_risk.json` | 市場危険度スコア（100点満点） |

## 2. 基本思想：情報起点のアプローチ

### 根本原則

```
投資 = 情報戦

情報は不確実性を減少させるものである。
1. 情報の確度を極限まで高める — 噂と事実を峻別し、一次資料で検証
2. 情報の非対称性を突く — 他者が知らない情報で優位に立つ
3. 情報の鮮度を管理する — 古い情報に基づく判断を排除する
```

### 情報起点 vs 企業起点

```
✓ 情報起点（このシステム）:
  幅広い情報収集 → 情報から企業発見 → 企業深掘り → 評価

✗ 企業起点（避けるべき）:
  企業リスト → その企業の情報収集 → 評価
```

情報が先、企業が後。**情報から企業を発見する**アプローチで、既知のバイアスを排除する。

### マルチLLMアーキテクチャ

複数のLLMから情報を収集し、Claudeが統合・分析する:

```
┌─────────────────────────────────────────────────────────┐
│  外部LLM（情報収集係）                                   │
│  - Claude, Gemini, GPT-4, Grok 等                       │
│  - 2種類の情報を収集:                                    │
│    1. company_info/: 企業・独占・財務情報               │
│    2. broad_info/: 広域情報（市場・政策）               │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Claude（分析係）                                        │
│  - 全LLMの出力を読み込み・統合                           │
│  - 重複排除、矛盾解決、4次元評価                         │
│  - 4軸評価、Top30選出、レポート生成                      │
│  - HTML可視化（4象限マトリックス）                       │
└─────────────────────────────────────────────────────────┘
```

**データ保存先**:
- `data/intelligence/raw/company_info/[llm名]_cd.md`
- `data/intelligence/raw/broad_info/[llm名]_mi.md`

### 「何を買うか」の前に「今買うべきか」

**市場危険度分析（Step 4）→ 銘柄選定（Step 5）**の順序は絶対。

バフェットの教え：「価値ある株が見つからないときはキャッシュポジションのまま様子をみる」

市場危険度が高い時に優良株を買っても、全体の下落に巻き込まれる。

## 3. 情報収集の7カテゴリ

全ての分析は**幅広い情報収集**から始まる。

| # | カテゴリ | 収集内容 | 活用先 |
|---|----------|----------|--------|
| A | **国際情勢** | 地政学、政策、規制、外交、紛争 | 市場危険度、ドメイン評価 |
| B | **市場危険度シグナル** | 金融政策、レバレッジ、バリュエーション、VIX | 市場危険度分析 |
| C | **メガトレンド** | AI、脱炭素、人口動態、宇宙開発、安全保障 | ドメイン成長性、企業発掘 |
| D | **6ドメインの成長性** | 各ドメインの市場規模、成長率、投資動向 | 銘柄評価（シナジー） |
| E | **大型投資パイプライン** | 政府予算、企業CAPEX、インフラ計画 | 企業発掘、独占力評価 |
| F | **見逃しスキャン** | 6ドメイン外の新興領域、技術ブレイクスルー | 新規ドメイン候補 |
| G | **その他** | 為替予測、金利見通し、センチメント | 総合判断 |

**全ての収集情報に4次元評価（確度・検証度・非対称性・鮮度）を付与する。**

## 4. 情報の4次元評価フレームワーク

全ての情報を以下の4軸で評価する。**確度の低い情報に基づく投資判断は、判断しないより悪い。**

### 4.1 確度（Certainty）— その情報はどの段階か

| ステージ | スコア | 定義 | 例 |
|----------|--------|------|-----|
| `rumor` | 10-20 | 噂、観測記事、関係者談 | 「○○を検討か」 |
| `report_unconfirmed` | 30-40 | 報道あるが一次資料未確認 | 「○○する方針」 |
| `report_confirmed` | 50-60 | 複数報道で確認、一次資料なし | 「○○で調整」 |
| `official_announcement` | 70-80 | 公式発表あり | 「○○を発表」 |
| `contract_signed` | 85-95 | 契約締結、予算成立、法案成立 | 「○○で合意」 |
| `executed` | 95-99 | 実行済み、納品済み、決算で確定 | 「○○が完了」 |

### 4.2 検証度（Verification）— どこまで裏取りしたか

| レベル | 定義 | 単独採用 |
|--------|------|----------|
| `primary_confirmed` | 一次資料で確認 | ✓ 可 |
| `multi_source` | 独立した2ソース以上 | ✓ 可 |
| `single_reliable` | 単一の信頼ソース | △ 条件付き |
| `single_unverified` | 単一・未検証 | ✗ 不可 |
| `conflicting` | ソース間で矛盾 | ✗ 不可 |

### 4.3 非対称性（Asymmetry）— 誰が知っているか

| スコア | 状態 | 説明 |
|--------|------|------|
| 21-25 | 極めて高い | 3源泉全てで非対称性あり |
| 16-20 | 高い | 2源泉で非対称性 |
| 11-15 | 中程度 | 1源泉で非対称性 |
| 6-10 | 低い | 一部グローバル投資家も認知 |
| 0-5 | なし | 広く認知済み |

### 4.4 鮮度（Freshness）— いつまで有効か

| 情報タイプ | 有効期間目安 |
|------------|--------------|
| 株価・為替 | 当日のみ |
| 需給 | 1週間 |
| 政策発表 | 実行まで |
| 世界シェア | 6ヶ月〜1年 |
| 認証・参入障壁 | 数年 |

## 5. 銘柄評価：4軸スコアリング

Top30の選出は以下の4軸で評価（**100点満点**）：

### 概要

| 軸 | 配点 | 内訳 |
|----|------|------|
| **独占力** | 30点 | 現在シェア(10点) + 5-10年持続性(10点) + 参入障壁(10点) |
| **資本効率** | 20点 | ROE(8点) + 負債比率(6点) + FCF(6点) ※事実ベースのみ |
| **シナジー** | 25点 | ドメイン横断不可欠性(15点) + 収斂価値増幅(10点) |
| **非対称性** | 25点 | 情報格差存在(10点) + 情報拡散時アップサイド(15点) |

---

### 5.1 独占力（30点）— 5-10年の見通しを含む

現時点のシェアだけでなく、**将来の競争優位持続性**を評価する。

| 評価項目 | 配点 | 内容 |
|----------|------|------|
| 現在のシェア・地位 | 10点 | 世界シェア、国内シェア |
| **5-10年の持続性** | 10点 | 将来の契約パイプライン、競合の開発状況 |
| 参入障壁の強度 | 10点 | 認証期間、技術蓄積年数、スイッチングコスト |

**ポイント**: 「今独占している」だけでなく「5-10年後も独占できるか」を評価。

---

### 5.2 資本効率（20点）— 事実ベースで評価

**取得可能な正確な情報**のみで評価。推定値・アナリスト予想は使わない。

| 評価項目 | 配点 | 基準（バフェット基準） |
|----------|------|------------------------|
| ROE | 8点 | 15%以上で満点 |
| 負債・資本比率 | 6点 | 33%以下で満点 |
| FCF（継続性） | 6点 | 5年連続プラスで満点 |

**データソース**: 有価証券報告書（EDINET）、決算短信、企業IR資料のみ。

---

### 5.3 シナジー（25点）— ドメイン横断の「不可欠性」

**「その技術がないと成り立たない」ドメインがいくつあるか**を評価。

#### イーロン・マスク型の統合モデル（参考）

```
インターネットインフラ ←→ AI事業 ←→ エネルギー事業
        ↓                   ↓              ↓
    宇宙事業            ロボティクス     蓄電池
        ↓                   ↓              ↓
    防衛・軍事          自動車（EV）     太陽光

→ どこでも必要不可欠な技術を持つ企業は圧倒的な優位性を持つ
```

| 評価項目 | 配点 | 内容 |
|----------|------|------|
| ドメイン横断の不可欠性 | 15点 | 不可欠なドメイン数（5-6で13-15点） |
| 収斂による価値増幅 | 10点 | ドメイン間の相乗効果 |

**6ドメイン**: AI、エネルギー、軍事・防衛、ロボティクス、宇宙、サイバー

---

### 5.4 非対称性（25点）— 情報拡散時のアップサイド

独占力・資本効率で測れない**「気づかれていない価値」**を評価。

ファンダメンタルズ成長ではなく、**情報格差が解消されたときのマルチプル拡大**を狙う。

| 評価項目 | 配点 | 内容 |
|----------|------|------|
| 情報格差の存在 | 10点 | 3源泉（言語、SC深層、認証）での非対称性 |
| **情報拡散時のアップサイド** | 15点 | 「気づかれたら」どれだけ評価が変わるか |

#### プロの見逃しを狙うポイント

- **食品/日用品企業の半導体材料** — セクター分類で見落とされる
- **地方の中小型株** — アナリストカバレッジなし
- **Tier3サプライヤー** — 最終製品からの可視性ゼロ
- **認証参入障壁** — 定量化しにくいため過小評価

---

### 4象限マトリックス

銘柄を「確度」と「非対称性」の2軸で分類：

```
              非対称性 高(16+)
                  ↑
    ┌─────────────┼─────────────┐
    │ 【要検証】    │ 【最優先】    │
    │ 確度<70     │ 確度≥70     │
    │ 追加調査で   │ 即座に       │
    │ 確度を上げる │ ポジション構築 │
確度├─────────────┼─────────────┤確度
<70 │ 【様子見】   │ 【安定】     │ ≥70
    │ 情報不足    │ αは小さいが  │
    │ 判断保留    │ 確実性高い   │
    └─────────────┴─────────────┘
              非対称性 低(<16)
```

## 6. 市場危険度分析

### 目的

「今買うべきか」を判断する。100点満点でスコアリング。

### Tier別重みづけ（歴史的破壊力に基づく）

| Tier | 項目 | 配点 | 根拠 |
|------|------|------|------|
| **Tier1** | 金融政策転換リスク | 15点 | 1990年バブル崩壊、2024年8月暴落 |
| **Tier1** | レバレッジ崩壊リスク | 15点 | 円キャリー巻き戻し |
| **Tier2** | バリュエーション過熱 | 10点 | PER/PBR異常高騰 |
| **Tier2** | VIX・ボラティリティ | 10点 | 低位安定は逆に警戒 |
| **Tier2** | 外国人売買動向 | 10点 | 需給の先行指標 |
| **Tier3** | 中国・新興国リスク | 10点 | 中国減速、通貨危機 |
| **Tier3** | 地政学リスク | 10点 | 台湾海峡、テロ等 |
| **Tier3** | 市場流動性 | 10点 | 売りが売りを呼ぶ |
| **Tier3** | 為替リスク | 10点 | 円高/円安の影響 |

**均等配分は怠慢**。歴史的に最も破壊的だった金融政策転換とレバレッジ崩壊に高い配点。

### バフェット的投資指針

| 市場危険度 | 解釈 | 推奨 |
|------------|------|------|
| 0-25 | 積極投資の好機 | フルポジション |
| 26-45 | 優良株を選別 | 通常ポジション |
| 46-60 | やや慎重姿勢 | 60-80%投資 |
| 61-75 | キャッシュを増やす | ポジション縮小 |
| 76-100 | 様子見、現金化 | 大幅縮小 |

## 7. 分析フロー（Step 0〜6）

### 全体フロー

```
Step 0: 広域情報収集（7カテゴリ） ← ★情報起点の核心（情報が先）
    │ 全情報に4次元評価を付与
    │ 情報から企業候補を発見
    ↓
Step 0.5: 企業発掘（70-100社）
    │ 認証リスト法、サプライチェーン逆引き法、ボトルネック・ニュース法
    ↓ 一次スクリーニング
候補プール（50社）
    ↓
Step 1: 企業別イベント収集
    ↓
Step 1.5: 情報品質チェック（矛盾・陳腐化検知）
    ↓
Step 2: 独占マップ更新
    ↓
Step 3: 市場状態取得
    ↓
Step 4: 市場危険度分析 ← 「今買うべきか」を判断
    ↓
Step 5: Top30スコアリング（4軸評価・100点満点）
    │
    ├→ Top30選出
    └→ 監視リスト(31-50位)
    ↓
Step 6: 最終レポート生成
    ↓
Step 7: HTML可視化生成 ← ★4象限マトリックス（確実性×アップサイド）
```

### 各Stepの詳細

| Step | 処理内容 | 出力 |
|------|----------|------|
| **Step 0** | 7カテゴリの広域情報収集、4次元評価付与 | `data/intelligence/` |
| **Step 0.5** | 70-100社を発掘、50社に絞り込み | `data/discovery/` |
| **Step 1** | 候補50社のイベント収集 | `data/events.json` |
| **Step 1.5** | 矛盾検知、陳腐化検知 | `data/information_quality.json` |
| **Step 2** | 独占マップ更新 | `data/snapshots/monopoly_map.json` |
| **Step 3** | 動的データ取得 | `data/snapshots/market_state.json` |
| **Step 4** | 市場危険度スコアリング | `data/analysis/market_risk.json` |
| **Step 5** | 4軸評価でTop30選出 | `data/analysis/top30.json` |
| **Step 6** | 最終レポート生成 | `reports/[日付]/final_report.md` |
| **Step 7** | HTML可視化生成 | `reports/[日付]/quadrant_chart.html`, `latest/` |

## 8. 使い方

### クイックスタート

```bash
# 1. リポジトリをクローン
git clone https://github.com/TakanoriISHII/japan-investment-analysis.git
cd japan-investment-analysis

# 2. Claude Codeで開く
claude

# 3. 分析を実行
> フル分析を実行
```

### コマンド一覧

| コマンド | 動作 |
|----------|------|
| `フル分析を実行` | 状態を確認し、必要な部分のみ更新して分析 |
| `強制フル更新` | 全てをゼロから再実行 |
| `広域情報収集` | Step 0のみ実行（7カテゴリ） |
| `企業発掘を実行` | Step 0.5のみ実行 |
| `情報品質を確認` | 未検証・矛盾・陳腐化アラートを表示 |
| `非対称性を確認` | アクティブな非対称性と解消リスクを表示 |
| `状態を確認` | 各コンポーネントの最終更新日時を表示 |
| `差分レポート` | 前回からの変化点のみ表示 |

### 最新結果の確認

```bash
# 最終レポート
cat latest/final_report.md

# 市場危険度
cat data/analysis/market_risk.json

# Top30ランキング
cat data/analysis/top30.json
```

## 10. 設計原則

### 絶対に変更しない原則

1. **情報起点のアプローチ**
   - 情報収集が先、企業選定が後

2. **「今買うべきか」→「何を買うか」の順序**
   - 市場危険度分析（Step 4）が先、銘柄選定（Step 5）が後

3. **4軸評価の定義**
   - 独占力：5-10年の持続性を含む
   - 資本効率：事実ベースのみ（推定値禁止）
   - シナジー：ドメイン横断の「不可欠性」
   - 非対称性：情報拡散時のアップサイド

4. **重みづけの根拠を明記**
   - 歴史的破壊力に基づく配点（均等配分は怠慢）

5. **情報品質フレームワーク**
   - 確度・検証度・非対称性・鮮度の4次元評価を維持

6. **非対称性の3源泉**
   - 言語、サプライチェーン深層、認証を常に意識
