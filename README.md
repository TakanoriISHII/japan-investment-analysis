# 日本マルチドメイン・コンバージェンス分析

> **フレームワーク**: Graham分類体系統合版 v2.0.0
> **対象市場**: 日本（東証上場企業中心）

## 概要

マルチドメイン・コンバージェンス投資分析フレームワークの日本版。6つの収束ドメインにまたがるボトルネック技術を支配する日本上場企業を特定し、Graham-Buffettの投資哲学に基づく投資的要因・投機的要因の分離評価を行う。

## 6つのドメイン

| # | ドメイン | 主要領域 |
|---|---------|---------|
| 1 | **AI** | コンピュートインフラ、半導体、データセンター |
| 2 | **エネルギー** | 原子力（SMR、HALEU）、グリッドインフラ、変圧器、蓄電 |
| 3 | **防衛** | ミサイル、航空機、艦船、弾薬、防衛エレクトロニクス |
| 4 | **ロボティクス** | 産業オートメーション、ドローン、UUV、ヒューマノイド |
| 5 | **宇宙** | 打上げ、衛星、月面、ISR |
| 6 | **サイバー** | 暗号化、電子戦、ISR融合、量子暗号 |

## 核心テーゼ

**マルチドメイン・コンバージェンス・ギャップ**: 市場は企業を単一セクター内で評価し、クロスドメインのシナジーを見逃している。複数ドメインにサービスを提供するボトルネック技術を支配する企業は、複合的な追い風の恩恵を受ける — しかしこの価値はしばしば認識されていない。

```
間違い: 「この企業はどのドメインに関与しているか？」
正しい: 「この技術がなくなったら何が止まるか？」
```

## 評価フレームワーク（Graham分類体系統合版 / 100点満点）

5軸は**投資的要因**と**投機的要因**に分離して評価する:

| 分類 | 軸 | 配点 | 説明 |
|------|-----|------|------|
| **B. 投資的要因** | 構造的優位性 | 20 | 市場シェア、参入障壁、モート財務裏付け |
| **(40点)** | 本質的価値 | 20 | 収益力、資産・資本構成、バリュエーション適正度 |
| **A. 投機的要因** | 将来の価値 | 25 | クロスドメイン・シナジー、成長確度、経営能力 |
| **(60点)** | 情報非対称性 | 20 | 市場認識ギャップ、全体性プレミアム、認識トリガー |
| | 政策・触媒 | 15 | 政策アクセラレーション、外部カタリスト |

**評価手順**: ベーススコア → 相対調整 → 情報確度(A-E)適用 → 最終スコア

### 情報確度フレームワーク

| レベル | 説明 | スコア乗数 |
|--------|------|-----------|
| A | 一次情報源で確認済み | 100% |
| B | 複数の信頼できる二次情報源 | 90% |
| C | 単一ソースまたは間接的確認 | 75% |
| D | 推定・計算値 | 60% |
| E | 未確認・噂レベル | 40% |

### 日本市場固有の基準

- **バリュエーション基準**: TSEセクター中央値PER（全体中央値 ~17倍）、PBR 1.4倍
- **財務健全性**: 自己資本比率 >50%、FCF黒字、有利子負債/EBITDA <2.0倍
- **利上げ局面調整**: 日銀政策転換期における財務健全性重視（軸2: 本質的価値で評価）
- **GNT認定**: 経産省グローバルニッチトップ企業認定の有無を構造的優位性に反映

## 3LLM体制

3つのLLMの特性を活かした役割分担で情報収集・分析を実行:

| LLM | 役割 | 主要タスク |
|-----|------|-----------|
| **Grok** | 速度重視・初期収集 | 広域情報スキャン、企業発掘、大量補強タスク |
| **Gemini** | 深さ重視・初期収集 | 企業深掘り分析、詳細調査、Graham-Buffett指標 |
| **Claude** | 論理推論・検証 | 矛盾解消、一次情報源確認、統合・評価・レポート |

## クイックスタート

### 1. インテリジェンス収集（3LLM並列実行）

`prompts/` のプロンプトをGrok・Geminiで実行:

| LLM | プロンプト | 目的 | 保存先 |
|-----|-----------|------|--------|
| Grok | `grok/company_scan.md` | 企業発掘 + Graham-Buffett指標 | `raw/grok/` |
| Grok | `grok/broad_info.md` | 広域情報（9カテゴリ） | `raw/grok/` |
| Gemini | `gemini/company_deep.md` | 企業深掘り + バリュエーション分析 | `raw/gemini/` |
| Gemini | `gemini/broad_info.md` | 広域情報（9カテゴリ、詳細版） | `raw/gemini/` |

出力を `data/intelligence/raw/[llm名]/[type]_YYYYMMDD.md` に保存

### 2. 分析実行

Claudeに伝える: `run analysis`

9フェーズが自動実行される:

| Phase | 内容 | 出力 |
|-------|------|------|
| 1 | 状態確認 + フレームワーク整合性検証 | PASS/WARN/HALT |
| 2 | インテリジェンス収集（3LLM） | `raw/grok/`, `raw/gemini/` |
| 3 | 比較分析（矛盾検出） | `contradictions.json`, `gaps.json` |
| 4 | 補足調査（Claude） | `claude_supplemental.md` |
| 5 | 統合・確度整理・stable更新 | `consolidated_*.json`, `stable/*` |
| 6 | 分析・評価（5軸 + IF/SF分離） | `market_risk.json`, `top30.json` |
| 7 | レポート生成 | `reports/[DATE]/` |
| 8 | 継続的改善 | `reinforcement/` |
| 9 | 完了・フレームワーク検証 | `state.json` 更新 |

### 3. 出力確認

`latest/` を確認:
- `final_report.md` — 完全分析レポート
- `executive_summary.md` — 主要な要点
- `information_quality_report.md` — 情報品質レポート
- `quadrant_chart.html` — 4象限チャート（IF/SF表示対応）

## 4象限マトリックス

| 象限 | 確実性 | アップサイド | アクション |
|------|--------|-------------|----------|
| 最優先 | 高 | 高 | コアポジション |
| 安定成長 | 高 | 低 | 分散保有 |
| 投機的 | 低 | 高 | 小ポジション、カタリスト待ち |
| 回避 | 低 | 低 | ポジションなし |

市場リスクスコアがポジショニングの積極性を決定:
- 低リスク (0-25): 非対称性を最大化、不確実性を許容
- 中リスク (26-50): 確実性とアップサイドのバランス
- 高リスク (51+): 確実性を優先、投機的ポジションを縮小

## 日本市場固有の特徴

- **非上場競合の構造**: 日本は「公開企業の非公開部門」が多い（防衛、宇宙、原子力等）
- **3LLM体制**: ChatGPTを除外し、Grok/Gemini/Claudeの3体制で運用
- **一次情報源**: TDnet（適時開示）、EDINET（有価証券報告書）、e-Gov（政府調達）、官報
- **GNT企業**: 経産省グローバルニッチトップ認定企業の構造的優位性を重視
- **利上げ局面**: 日銀政策転換期における財務健全性（自己資本比率、FCF、有利子負債比率）を評価に反映
