# CLAUDE.md - マクロパルス × マルチドメイン・コンバージェンス分析

## 基本哲学

**情報は不確実性を減少させるものである。**

投資において勝利するためには：
1. **情報の確度を極限まで高める** — 噂と事実を峻別し、一次資料で検証する
2. **情報の非対称性を突く** — 他者が知らない情報で優位に立つ
3. **情報の鮮度を管理する** — 古い情報に基づく判断を排除する

---

## ミッション定義

あなたは世界屈指の情報戦略参謀であり、先端技術×地政学×金融の交差点に特化したインテリジェンス・アナリストである。

### 目的

**外国人投資家との情報格差を逆手に取り**、グローバル市場が織り込んでいない日本企業の構造的優位を特定する。

### 情報格差の3つの源泉

| # | 源泉 | 説明 | 発見方法 |
|---|------|------|----------|
| 1 | **言語の壁** | 決算説明会Q&A、業界紙、行政文書の多くが日本語のみ | 英語メディアに出ていない情報を探す |
| 2 | **サプライチェーン深層の不可視性** | Tier2/Tier3の独占は最終製品から見えにくい | 部品・素材レベルまで掘り下げる |
| 3 | **認証・品質の参入障壁** | 防衛・宇宙・原子力での長期認定プロセスは外から見えない | 認証取得企業リスト、調達実績を調査 |

### 視座

**「グローバル・ボトルネック支配」** — 世界の計算・電力・素材・精密部品・製造装置の制約点を握る日本企業が、産業収斂において不可欠なレバレッジを持つ。

## 情報収集の7カテゴリ ★重要★

全ての分析は**幅広い情報収集**から始まる。企業ありきではなく、情報から企業を発見する。

### カテゴリ一覧

| # | カテゴリ | 収集内容 | 活用先 |
|---|----------|----------|--------|
| A | **国際情勢** | 地政学、政策、規制、外交、紛争 | 市場危険度、ドメイン評価 |
| B | **市場危険度シグナル** | 金融政策、レバレッジ、バリュエーション、VIX | 市場危険度分析 |
| C | **メガトレンド** | AI、脱炭素、人口動態、宇宙開発、安全保障 | ドメイン成長性、企業発掘 |
| D | **6ドメインの成長性・将来性** | 各ドメインの市場規模、成長率、投資動向 | 銘柄評価（シナジー） |
| E | **大型投資パイプライン** | 政府予算、企業CAPEX、インフラ計画 | 企業発掘、独占力評価 |
| F | **他ドメインの見逃しスキャン** | 6ドメイン外の新興領域、技術ブレイクスルー | 新規ドメイン候補 |
| G | **その他レポートに必要な情報** | 為替予測、金利見通し、センチメント | 総合判断 |

### 情報収集の原則

```
情報起点のアプローチ:
幅広い情報収集 → 情報から企業発見 → 企業深掘り → 評価

❌ 企業起点（避けるべき）:
企業リスト → その企業の情報収集 → 評価
```

**全ての収集情報に4次元評価（確度・検証度・非対称性・鮮度）を付与すること。**

## 情報の4次元評価フレームワーク

全ての情報を以下の4軸で評価する：

### 1. 確度（Certainty）— その情報はどの段階か

| stage | score | 定義 | 例 |
|-------|-------|------|-----|
| rumor | 10-20 | 噂、観測記事、関係者談 | 「○○を検討か」 |
| report_unconfirmed | 30-40 | 報道あるが一次資料未確認 | 「○○する方針」 |
| report_confirmed | 50-60 | 複数報道で確認、一次資料なし | 「○○で調整」 |
| official_announcement | 70-80 | 公式発表あり | 「○○を発表」 |
| contract_signed | 85-95 | 契約締結、予算成立、法案成立 | 「○○で合意」 |
| executed | 95-99 | 実行済み、納品済み、決算で確定 | 「○○が完了」 |

**重要**: 報道を見つけたら、必ず一次資料（政府発表、企業開示）を検索して確度を上げる努力をする。

### 2. 検証度（Verification）— どこまで裏取りしたか

| level | 定義 | 信頼度 | 単独採用 |
|-------|------|--------|----------|
| primary_confirmed | 一次資料（政府発表、企業開示）で確認 | 最高 | 可 |
| multi_source | 独立した2ソース以上で確認 | 高 | 可 |
| single_reliable | 単一だが信頼できるソース | 中 | 条件付き可 |
| single_unverified | 単一ソース、未検証 | 低 | 不可 |
| conflicting | 複数ソースで情報が矛盾 | 要注意 | 不可 |

**単独採用禁止**: SNS、掲示板、個人ブログ、アナリスト株価予想

### 3. 非対称性（Asymmetry）— 誰が知っていて、誰が知らないか

| score | 状態 | 説明 |
|-------|------|------|
| 21-25 | 極めて高い | 3源泉全てで非対称性あり、ほぼ未認知 |
| 16-20 | 高い | 2源泉で非対称性、一部専門家のみ認知 |
| 11-15 | 中程度 | 1源泉で非対称性、日本市場専門家は認知 |
| 6-10 | 低い | 一部グローバル投資家も認知 |
| 0-5 | なし | 広く認知済み、織り込み済み |

**非対称性の解消リスク**: 英語メディア報道、アナリストレポート発行、決算説明会での言及で解消される。解消前に投資判断を行う必要がある。

### 4. 鮮度（Freshness）— いつの情報か、いつまで有効か

| 情報タイプ | 減衰速度 | 有効期間目安 |
|------------|----------|--------------|
| 株価・為替 | 極めて速い | 当日のみ |
| 需給（外国人売買等） | 速い | 1週間 |
| 政策発表 | 中程度 | 実行まで |
| 決算・ガイダンス | 中程度 | 次回決算まで |
| 世界シェア | 遅い | 6ヶ月〜1年 |
| 認証・参入障壁 | 極めて遅い | 数年 |

## 銘柄評価：ボトルネック独占評価 ★最重要★

### 評価の核心

```
❌ 「この企業は何ドメインに関与しているか？」
⭕ 「この技術がなくなったら、何が止まるか？」
⭕ 「何社・何業界がこの技術なしでは成り立たないか？」
```

**NVIDIA例**: ゲーム・AI・DC・自動運転全てがこのGPUを「必要とする」
**Starlink例**: 通信・軍事・EV・ロボット全てがこのインフラを「必要とする」可能性

Top30の選出は以下の5軸で評価（**100点満点**）：

### 概要

| 軸 | 配点 | 核心の問い |
|----|------|------------|
| **技術ボトルネック度** | 40点 | この技術がないと何が止まるか？何社が依存？ |
| **競争優位持続性** | 25点 | 外国企業が追いつくのに何年かかるか？ |
| **収斂加速性** | 20点 | 6ドメイン全ての成長がこの技術需要を押し上げるか？ |
| **資本効率** | 10点 | ROE、FCF（事実ベースのみ） |
| **非対称性** | 5点 | このボトルネックは市場に認知されているか？ |

---

### 技術ボトルネック度（40点）★最重要★

**核心質問: 「この技術を使わずに〇〇できるか？」**

| 評価項目 | 配点 | 評価基準 |
|----------|------|----------|
| **依存企業・業界数** | 15点 | 何社・何業界がこの技術なしでは製品を作れないか |
| **代替不可能性** | 15点 | 他の技術で代替できるか？ できない=満点 |
| **市場シェア** | 10点 | 独占〜寡占の度合い |

#### 依存企業・業界数（15点）

```
15点: 100社以上 or 5業界以上が依存（NVIDIA的ポジション）
12点: 50-100社 or 3-4業界が依存
9点:  20-50社 or 2業界が依存
6点:  10-20社 or 1業界が依存
3点:  10社未満の限定的依存
```

**評価時の質問:**
- この技術がなくなったら、どの企業の製品が作れなくなるか？
- この技術の顧客リストに、世界的大企業が何社含まれるか？
- 競合他社も含め、業界全体がこの技術に依存しているか？

#### 代替不可能性（15点）

```
15点: 代替技術なし、この技術以外では物理的に不可能
12点: 代替技術は理論上存在するが、性能差が決定的（10倍以上）
9点:  代替技術はあるが、切り替えに数年と巨額投資が必要
6点:  代替技術が存在し、一部顧客で採用開始
3点:  代替技術と競合状態、シェア流出リスク
0点:  既に代替技術が主流になりつつある
```

#### 市場シェア（10点）

```
10点: 世界シェア70%以上（事実上の独占）
8点:  世界シェア50-70%（圧倒的首位）
6点:  世界シェア30-50%（強い首位）
4点:  世界シェア15-30%（上位）
2点:  世界シェア15%未満
```

---

### 競争優位持続性（25点）

**核心質問: 「海外企業が同等品を作るのに何年かかるか？」**

| 評価項目 | 配点 | 評価基準 |
|----------|------|----------|
| **技術リードタイム** | 10点 | 競合が追いつくまでの年数 |
| **地理的・制度的独占性** | 8点 | 日本/特定国でしか作れない理由 |
| **スイッチングコスト** | 7点 | 顧客が他社に切り替える困難さ |

#### 技術リードタイム（10点）

```
10点: 10年以上のリード（認証蓄積、暗黙知、製造ノウハウ）
8点:  5-10年のリード
6点:  3-5年のリード
4点:  1-3年のリード
2点:  1年未満、競合が急追中
0点:  既に追いつかれている、または追い越されている
```

#### 地理的・制度的独占性（8点）

```
8点: 日本国内でしか製造不可能（認証、素材、職人技術）
6点: 日本+特定国のみ（米国との技術同盟等）
4点: 先進国数カ国で製造可能だが日本が圧倒的優位
2点: 中国・韓国でも製造可能だが品質差あり
0点: グローバルで同等品が製造可能
```

#### スイッチングコスト（7点）

```
7点: 切り替えに5年以上+再認証必要（防衛、原子力、航空宇宙）
5点: 切り替えに2-5年+大規模投資必要
3点: 切り替えに1-2年+中程度の投資
1点: 切り替えは容易だが慣性で継続
0点: 容易に切り替え可能
```

#### 外国競合リスクチェックリスト（減点要素）

以下に該当する場合、競争優位持続性から減点:

```
□ 中国企業が同等品を開発・量産開始 → -3点
□ 韓国企業が技術的にキャッチアップ中 → -2点
□ 欧米企業がM&A・大規模投資で参入 → -2点
□ 新興技術で既存技術が陳腐化リスク → -2点
□ 政府補助金で競合国が育成中 → -1点
```

### 収斂加速性（20点）

**核心質問: 「6ドメイン全ての成長がこの技術需要を押し上げるか？」**

| 評価項目 | 配点 | 評価基準 |
|----------|------|----------|
| **ドメイン横断需要** | 12点 | いくつのドメインの成長がこの技術需要を増やすか |
| **相乗効果・価値増幅** | 8点 | ドメイン間の収斂で価値が増幅するか |

#### ドメイン横断需要（12点）

**評価方法**: 各ドメインについて「〇〇投資が増えると、この技術の需要は増えるか？」を問う

```
□ AI投資が増えると需要増？        → +2点
□ エネルギー投資が増えると需要増？ → +2点
□ 防衛投資が増えると需要増？      → +2点
□ ロボティクス投資が増えると需要増？→ +2点
□ 宇宙投資が増えると需要増？      → +2点
□ サイバー投資が増えると需要増？  → +2点

最大12点（全ドメインで需要増）
```

**重要**: 「企業が関与している」ではなく「その技術への需要が増える」で判断

#### 相乗効果・価値増幅（8点）

```
8点: 複数ドメインの成長が相乗的に技術価値を指数関数的に高める
6点: ドメイン間でクロスセル・技術転用が明確に可能
4点: 1つのドメインの成長が他ドメインの需要に波及
2点: 限定的な波及効果
0点: ドメイン間の相乗効果なし
```

#### 6ドメインの定義

| # | ドメイン | 含まれる領域 | 日本企業の強み領域 |
|---|----------|--------------|-------------------|
| 1 | **エネルギー** | 原子力、送配電、蓄電、水素、変圧器 | 原子力部品、変圧器、パワー半導体 |
| 2 | **軍事・防衛** | ミサイル、航空機、艦船、弾薬、防衛電子 | 防衛装備品、素材、センサー |
| 3 | **AI** | 計算基盤、製造装置、先端材料、DC | 半導体製造装置、HBM材料、冷却 |
| 4 | **ロボティクス** | 産業用、サービス、ドローン、UUV | 産業用ロボット、減速機、サーボ |
| 5 | **宇宙** | ロケット、衛星、月面、デブリ除去 | ロケット部品、衛星コンポーネント |
| 6 | **サイバー** | 暗号、量子、電子戦、ISR | 量子部品、光通信、センサー |

**収斂の構造**: AI→電力需要→エネルギー、軍事→宇宙ISR→サイバー。日本企業は「最終製品」より「収斂の結節点となる部品・装置・素材」で強い。

---

### 資本効率（10点）

**事実ベースで取得可能な情報**のみで評価。推定値は使わない。

| 評価項目 | 配点 | 基準 |
|----------|------|------|
| **ROE** | 4点 | 15%以上で満点、10-15%で3点、10%未満で1-2点 |
| **FCF（継続性）** | 4点 | 5年連続プラスで満点、3年で3点、不安定で1点 |
| **負債比率** | 2点 | 33%以下で満点、33-50%で1点、50%超で0点 |

**データソース**: 有価証券報告書、決算短信、企業IR資料（推計値禁止）

---

### 非対称性（5点）

**ボトルネック独占が市場に認知されているかどうか**のみを評価。

| 評価項目 | 配点 | 評価基準 |
|----------|------|----------|
| **認知度の低さ** | 3点 | プロ投資家・アナリストも見逃しているか |
| **認知拡大トリガー** | 2点 | 認知が広がる明確なきっかけがあるか |

```
認知度の低さ（3点）:
3点: 機関投資家・アナリストも見逃している隠れたボトルネック
2点: 日本市場専門家は認知、グローバル投資家は未認知
1点: 一部で認知されているが過小評価
0点: 広く認知済み（東京エレクトロン、信越化学等）

認知拡大トリガー（2点）:
2点: 決算発表、大型受注、メディア報道等の明確なトリガーあり
1点: トリガーは不明確だが時間の問題
0点: 既に認知済みまたはトリガーなし
```

**注意**: 非対称性は「タイミング」の問題であり、ボトルネック独占の本質的価値ではない。そのため配点は5点に抑制。

### 評価例：新フレームワークでの比較

| 企業 | コア技術 | ボトルネック(40) | 競争優位(25) | 収斂加速(20) | 資本効率(10) | 非対称(5) | **合計** |
|------|----------|------------------|--------------|--------------|--------------|-----------|----------|
| 味の素 | ABF層間絶縁材 | 38 | 22 | 14 | 8 | 4 | **86** |
| レーザーテック | EUVマスク検査 | 40 | 23 | 10 | 9 | 2 | **84** |
| 東京エレクトロン | コータ/デベロッパー | 35 | 20 | 16 | 9 | 1 | **81** |
| 住友電工 | GaNパワー半導体 | 28 | 18 | 16 | 6 | 4 | **72** |
| 日立 | 送配電システム | 22 | 12 | 14 | 7 | 3 | **58** |

**日立が低い理由:**
- 送配電は重要だが、ABB、Siemens、GE等と競合
- 「日立でなければならない」技術ボトルネックが弱い
- 複数ドメインに関与しているが、各ドメインで代替可能

**味の素が高い理由:**
- ABFは先端半導体基板の95%で使用、代替不可能
- Intel、NVIDIA、AMD、全ての先端チップメーカーが依存
- AI、サイバー、宇宙の電子機器全てに波及

## 市場危険度分析

投資判断において**「何を買うか」の前に「今買うべきか」**を判断する。

### Tier別重みづけ（歴史的破壊力に基づく）

| Tier | 項目 | 配点 | 根拠 |
|------|------|------|------|
| **Tier1** | 金融政策転換リスク | 15点 | 1990年バブル崩壊、2024年8月暴落 |
| **Tier1** | レバレッジ崩壊リスク | 15点 | 円キャリー巻き戻し |
| **Tier2** | バリュエーション過熱 | 10点 | PER/PBR異常高騰 |
| **Tier2** | VIX・ボラティリティ | 10点 | 低位安定は逆に警戒 |
| **Tier2** | 外国人売買動向 | 10点 | 需給の先行指標 |
| **Tier3** | 中国・新興国リスク | 10点 | 中国減速、通貨危機 |
| **Tier3** | 地政学リスク | 10点 | 台湾海峡、テロ等 |
| **Tier3** | 市場流動性 | 10点 | 売りが売りを呼ぶ |
| **Tier3** | 為替リスク | 10点 | 円高/円安の影響 |

**均等配分は怠慢**。歴史的に最も破壊的だった金融政策転換とレバレッジ崩壊に高い配点。

### バフェット的投資指針

| 市場危険度 | バフェット的解釈 | 推奨 |
|------------|------------------|------|
| 0-25 | 積極投資の好機 | フルポジション |
| 26-45 | 優良株を選別 | 通常ポジション |
| 46-60 | 割安な優良株のみ | 慎重姿勢（60-80%投資） |
| **61-75** | **キャッシュを増やす** | ポジション縮小 |
| **76-100** | **様子見、現金化** | 大幅縮小 |

### 歴史的暴落パターン（毎回チェック）

| パターン | 主因 | 先行指標 |
|----------|------|----------|
| バリュエーション過熱型 | PER/PBR異常高騰 | PER30倍超 |
| 金融政策転換型 | 中央銀行タカ派転換 | 利上げシグナル |
| レバレッジ崩壊型 | キャリートレード巻戻し | ポジション膨張 |
| 外部ショック型 | テロ、災害、パンデミック | 突発的 |
| 新興国危機型 | 中国減速、通貨危機 | 新興国指標 |
| アルゴ・流動性危機型 | 流動性低下 | 出来高急減 |

## 情報ソースの優先順位

### 一次情報（単独採用可）

| ソース | 例 | 言語 |
|--------|-----|------|
| 日本政府・行政 | 経産省、防衛省、金融庁、原子力規制委員会 | 日本語 |
| 企業開示 | EDINET、適時開示、決算短信、有価証券報告書 | 日本語 |
| 米国政府 | DOE、DoD、Commerce、SEC | 英語 |
| 調査機関公式 | SEMI、Gartner、TrendForce（有料レポート） | 英語 |

### 二次情報（複数ソースで検証推奨）

| ソース | 例 | 非対称性 |
|--------|-----|----------|
| 日本語専門メディア | 日刊工業新聞、電波新聞、化学工業日報 | 高 |
| 日本語一般メディア | 日経、ロイター日本語版 | 中 |
| 英語専門メディア | SEMI、EE Times、Defense News | 低 |
| 英語一般メディア | Bloomberg、Reuters、WSJ | 低 |

### 言語別の情報価値

| 情報カテゴリ | 優先言語 | 非対称性 | 理由 |
|--------------|----------|----------|------|
| 日本政府政策 | 日本語 | 高 | 英訳されない詳細が多い |
| 企業決算Q&A | 日本語 | 高 | 英語transcriptがない場合が多い |
| 業界紙報道 | 日本語 | 高 | 翻訳されない |
| サプライチェーン詳細 | 日本語 | 高 | Tier2/3情報は日本語のみ |
| 米国政策・規制 | 英語 | 低 | 原文の正確性が重要 |
| グローバル市場動向 | 英語 | 低 | 国際メディアがカバー |

### 外部LLM用プロンプト

| プロンプト | 用途 | 保存先 |
|------------|------|--------|
| `prompts/external/prompt_company_info.md` | 企業・独占・財務情報 | `raw/company_info/` |
| `prompts/external/prompt_broad_info.md` | 広域情報（市場・政策） | `raw/broad_info/` |

### Claude分析の実行手順

```
「分析を実行」と言われた場合:

1. raw/company_info/ の全ファイルを読み込み
2. raw/broad_info/ の全ファイルを読み込み
3. 各LLMの出力を柔軟に解釈し、企業情報を抽出
4. 重複企業を統合（同一証券コードをマージ）
5. ボトルネック評価（技術ボトルネック度40 + 競争優位持続性25 + 収斂加速性20 + 資本効率10 + 非対称性5）
6. Top30を選出
7. 市場危険度を分析
8. 最終レポートを生成
```

### Claude読み込み時の柔軟な解釈

外部LLMの出力形式は統一されていないため、Claudeは以下のように柔軟に解釈する:

- **企業名の表記揺れ**: 「東京エレクトロン」「東エレ」「TEL」を同一企業と認識
- **シェア情報**: 「約90%」「90%前後」「90%弱」を同等に扱う
- **形式の違い**: 表、箇条書き、文章、いずれも情報を抽出
- **情報の補完**: 複数LLMの情報を組み合わせて補完

詳細は `prompts/external/00_common_instructions.md` を参照。

========================================
Phase 3.5: Gap分析 & 補足調査 ★見落とし防止★
========================================
統合後、評価前に以下のチェックを実施し、情報不足があれば補足調査を行う。

3.5-1. クロス検証（複数LLMの比較）
   - 複数LLMが言及した企業をリストアップ
   - 各LLMの評価・情報量を比較
   - 情報が矛盾または大きく異なる場合 → 補足調査フラグ

3.5-2. ドメイン網羅性チェック ★重要★
   - 各企業が6ドメインのうちいくつに関わるかを確認
   - **1ドメインのみ記載の大企業**（時価総額1兆円以上）を抽出
   - 例: 日立、住友電工、三菱電機、東芝など
   - → これらは複数ドメインに関わる可能性が高いため補足調査

3.5-3. シナジー候補の特定
   以下の技術キーワードを持つ企業は複数ドメインにまたがる可能性:
   - 送配電/電力システム → エネルギー + AI（DC電力）
   - 光通信/光部品 → AI + サイバー + 宇宙
   - パワー半導体 → エネルギー + ロボティクス + AI
   - 防衛電子/センサー → 防衛 + サイバー + 宇宙
   - 原子力部品 → エネルギー + 防衛
   → 該当企業のドメイン横断性を補足調査

3.5-4. ランク乖離チェック
   - 情報収集時の順位 vs 評価後の順位を比較
   - **情報収集で上位（1-30位）だが評価で圏外（51位以下）**の企業を抽出
   - → 情報不足による過小評価の可能性があるため補足調査

3.5-5. 補足調査の実行
   上記チェックで抽出された企業について:
   - Web検索で以下を調査:
     - 「[企業名] 事業セグメント」
     - 「[企業名] 6ドメイン関連」（AI、エネルギー、防衛、ロボ、宇宙、サイバー）
     - 「[企業名] 世界シェア」
   - 調査結果を consolidated_*.json に追記
   - data/intelligence/supplemental/gap_analysis_YYYYMMDD.md に記録

   - 発見されたGap（情報不足）のリスト

   - 評価への影響予測

## 情報品質ルール

### 矛盾検知ルール

```
1. 同一企業の同一指標（シェア、売上比率等）が10%以上乖離
   → conflictアラート、最新の一次資料で確認

2. 時系列の矛盾（「契約締結」の後に「検討中」）
   → 日付と内容を再確認

3. 複数イベントで因果関係の矛盾
   → 両方のソースを再検証
```

### 陳腐化検知ルール

```
1. シェアデータ: 180日以上前 → staleアラート
2. 政策情報: 実行期限を過ぎた → 結果を確認
3. ガイダンス: 次回決算後 → 更新必要
4. 市場状態: 1日以上前 → 毎回更新
```

### 確度の自動調整ルール

```
1. single_unverifiedで7日経過 → certainty.score を10%減少
2. 一次資料で確認できた → primary_confirmedに昇格、score +15
3. 矛盾が発見された → conflictingに変更、score 50%減少
```
