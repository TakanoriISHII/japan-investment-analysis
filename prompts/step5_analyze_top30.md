# Step 5: Top30スコアリング（ボトルネック独占評価）

## ミッション

**候補プール50社**をボトルネック独占評価（100点満点）でランキングし、**Top30**を選出する。
31-50位は**監視リスト**として保持し、昇格候補として追跡する。

**核心の問い: 「この技術がなくなったら、何が止まるか？何社が依存しているか？」**

---

## ファネルの位置づけ

```
Step 0: 大量発掘（70-100社）→ 候補プール（50社）
    ↓
Step 5（本ステップ）:
    候補プール50社
        ↓ ボトルネック評価（100点満点）
    Top30選出 + 監視リスト(31-50位)
```

**重要**: Step 0では簡易スクリーニングのみ実施。Step 5で「この技術がなくなったら何が止まるか」を詳細に評価する。

---

## 評価の核心

```
❌ 「この企業は何ドメインに関与しているか？」
⭕ 「この技術がなくなったら、何が止まるか？」
⭕ 「何社・何業界がこの技術なしでは成り立たないか？」

NVIDIA例: ゲーム・AI・DC・自動運転全てがこのGPUを「必要とする」
Starlink例: 通信・軍事・EV・ロボット全てがこのインフラを「必要とする」可能性
```

## 評価フレームワーク（100点満点）★5軸定義

### 概要

| 軸 | 配点 | 核心の問い |
|----|------|------------|
| **技術ボトルネック度** | 40点 | この技術がないと何が止まるか？何社が依存？ |
| **競争優位持続性** | 25点 | 外国企業が追いつくのに何年かかるか？ |
| **収斂加速性** | 20点 | 6ドメイン全ての成長がこの技術需要を押し上げるか？ |
| **資本効率** | 10点 | ROE、FCF（事実ベースのみ） |
| **非対称性** | 5点 | このボトルネックは市場に認知されているか？ |

---

## 評価軸1: 技術ボトルネック度（40点満点）★最重要★

**核心質問: 「この技術を使わずに〇〇できるか？」**

| 評価項目 | 配点 | 評価基準 |
|----------|------|----------|
| **依存企業・業界数** | 15点 | 何社・何業界がこの技術なしでは製品を作れないか |
| **代替不可能性** | 15点 | 他の技術で代替できるか？ できない=満点 |
| **市場シェア** | 10点 | 独占〜寡占の度合い |

### 依存企業・業界数（15点）

```
15点: 100社以上 or 5業界以上が依存（NVIDIA的ポジション）
12点: 50-100社 or 3-4業界が依存
9点:  20-50社 or 2業界が依存
6点:  10-20社 or 1業界が依存
3点:  10社未満の限定的依存
```

**評価時の質問:**
- この技術がなくなったら、どの企業の製品が作れなくなるか？
- この技術の顧客リストに、世界的大企業が何社含まれるか？
- 競合他社も含め、業界全体がこの技術に依存しているか？

### 代替不可能性（15点）

```
15点: 代替技術なし、この技術以外では物理的に不可能
12点: 代替技術は理論上存在するが、性能差が決定的（10倍以上）
9点:  代替技術はあるが、切り替えに数年と巨額投資が必要
6点:  代替技術が存在し、一部顧客で採用開始
3点:  代替技術と競合状態、シェア流出リスク
0点:  既に代替技術が主流になりつつある
```

### 市場シェア（10点）

```
10点: 世界シェア70%以上（事実上の独占）
8点:  世界シェア50-70%（圧倒的首位）
6点:  世界シェア30-50%（強い首位）
4点:  世界シェア15-30%（上位）
2点:  世界シェア15%未満
```

---

## 評価軸2: 競争優位持続性（25点満点）

**核心質問: 「海外企業が同等品を作るのに何年かかるか？」**

| 評価項目 | 配点 | 評価基準 |
|----------|------|----------|
| **技術リードタイム** | 10点 | 競合が追いつくまでの年数 |
| **地理的・制度的独占性** | 8点 | 日本/特定国でしか作れない理由 |
| **スイッチングコスト** | 7点 | 顧客が他社に切り替える困難さ |

### 技術リードタイム（10点）

```
10点: 10年以上のリード（認証蓄積、暗黙知、製造ノウハウ）
8点:  5-10年のリード
6点:  3-5年のリード
4点:  1-3年のリード
2点:  1年未満、競合が急追中
0点:  既に追いつかれている、または追い越されている
```

### 地理的・制度的独占性（8点）

```
8点: 日本国内でしか製造不可能（認証、素材、職人技術）
6点: 日本+特定国のみ（米国との技術同盟等）
4点: 先進国数カ国で製造可能だが日本が圧倒的優位
2点: 中国・韓国でも製造可能だが品質差あり
0点: グローバルで同等品が製造可能
```

### スイッチングコスト（7点）

```
7点: 切り替えに5年以上+再認証必要（防衛、原子力、航空宇宙）
5点: 切り替えに2-5年+大規模投資必要
3点: 切り替えに1-2年+中程度の投資
1点: 切り替えは容易だが慣性で継続
0点: 容易に切り替え可能
```

### 外国競合リスクチェックリスト（減点要素）

以下に該当する場合、競争優位持続性から減点:

```
□ 中国企業が同等品を開発・量産開始 → -3点
□ 韓国企業が技術的にキャッチアップ中 → -2点
□ 欧米企業がM&A・大規模投資で参入 → -2点
□ 新興技術で既存技術が陳腐化リスク → -2点
□ 政府補助金で競合国が育成中 → -1点
```

## 評価軸3: 収斂加速性（20点満点）

**核心質問: 「6ドメイン全ての成長がこの技術需要を押し上げるか？」**

| 評価項目 | 配点 | 評価基準 |
|----------|------|----------|
| **ドメイン横断需要** | 12点 | いくつのドメインの成長がこの技術需要を増やすか |
| **相乗効果・価値増幅** | 8点 | ドメイン間の収斂で価値が増幅するか |

### ドメイン横断需要（12点）

**評価方法**: 各ドメインについて「〇〇投資が増えると、この技術の需要は増えるか？」を問う

```
□ AI投資が増えると需要増？        → +2点
□ エネルギー投資が増えると需要増？ → +2点
□ 防衛投資が増えると需要増？      → +2点
□ ロボティクス投資が増えると需要増？→ +2点
□ 宇宙投資が増えると需要増？      → +2点
□ サイバー投資が増えると需要増？  → +2点

最大12点（全ドメインで需要増）
```

**重要**: 「企業が関与している」ではなく「その技術への需要が増える」で判断

### 相乗効果・価値増幅（8点）

```
8点: 複数ドメインの成長が相乗的に技術価値を指数関数的に高める
6点: ドメイン間でクロスセル・技術転用が明確に可能
4点: 1つのドメインの成長が他ドメインの需要に波及
2点: 限定的な波及効果
0点: ドメイン間の相乗効果なし
```

### 6ドメイン定義

| # | ドメイン | 含まれる領域 |
|---|----------|--------------|
| 1 | エネルギー | 原子力、送配電、蓄電、水素、変圧器 |
| 2 | 軍事・防衛 | ミサイル、航空機、艦船、弾薬、防衛電子 |
| 3 | AI | 計算基盤、製造装置、先端材料、DC |
| 4 | ロボティクス | 産業用、サービス、ドローン、UUV |
| 5 | 宇宙 | ロケット、衛星、月面、デブリ除去 |
| 6 | サイバー | 暗号、量子、電子戦、ISR |

## 評価軸4: 資本効率（10点満点）

**事実ベースで取得可能な情報**のみで評価。推定値は使わない。

| 評価項目 | 配点 | 基準 |
|----------|------|------|
| **ROE** | 4点 | 15%以上で満点、10-15%で3点、10%未満で1-2点 |
| **FCF（継続性）** | 4点 | 5年連続プラスで満点、3年で3点、不安定で1点 |
| **負債比率** | 2点 | 33%以下で満点、33-50%で1点、50%超で0点 |

**データソース**: 有価証券報告書、決算短信、企業IR資料（推計値禁止）

---

## 評価軸5: 非対称性（5点満点）

**ボトルネック独占が市場に認知されているかどうか**のみを評価。

| 評価項目 | 配点 | 評価基準 |
|----------|------|----------|
| **認知度の低さ** | 3点 | プロ投資家・アナリストも見逃しているか |
| **認知拡大トリガー** | 2点 | 認知が広がる明確なきっかけがあるか |

```
認知度の低さ（3点）:
3点: 機関投資家・アナリストも見逃している隠れたボトルネック
2点: 日本市場専門家は認知、グローバル投資家は未認知
1点: 一部で認知されているが過小評価
0点: 広く認知済み（東京エレクトロン、信越化学等）

認知拡大トリガー（2点）:
2点: 決算発表、大型受注、メディア報道等の明確なトリガーあり
1点: トリガーは不明確だが時間の問題
0点: 既に認知済みまたはトリガーなし
```

**注意**: 非対称性は「タイミング」の問題であり、ボトルネック独占の本質的価値ではない。そのため配点は5点に抑制。

## 確実性（Certainty）の計算

各企業について、以下を総合評価：

| 要素 | 高確実性 | 低確実性 |
|------|----------|----------|
| ボトルネック度 | 代替不可能、依存企業多数 | 代替技術あり、依存限定的 |
| 競争優位持続性 | 10年以上のリード | 追いつかれるリスク大 |
| シェア安定性 | 過去5年で変動5%以内 | 変動大、競合台頭 |
| 外国競合リスク | 日本独自、追随困難 | 中国・韓国が急追 |
| 契約・バックログ | 長期契約確定 | 短期・スポット |

**計算式**:
```
certainty = 基礎点(50)
  + 技術ボトルネック度(0-20)  ← 40点満点を0-20にスケール
  + 競争優位持続性(0-15)      ← 25点満点を0-15にスケール
  + 外国競合リスク調整(-15〜+5)
  + シェア安定性(0-10)
```

**目安**:
- 90以上: 代替不可能なボトルネック、追随困難
- 80-89: 強いボトルネック、5年以上のリード
- 70-79: 中程度のボトルネック、競争あり
- 70未満: ボトルネック度が弱い、または競合リスク大

### アップサイド（Upside）の計算

収斂加速性スコア（/20）+ 非対称性（/5）を基準に計算：

```
upside = 基礎点(50)
  + 収斂加速性(0-20)  ← 20点をそのまま使用
  + 非対称性(0-5) × 3  ← 5点を0-15にスケール
  + ドメイン成長率調整(0-15)
```

---

## 4象限配置

```
if (certainty >= 80 && upside >= 75) → "prime"       // 最優先投資
if (certainty >= 80 && upside < 75)  → "stable"      // 安定成長
if (certainty < 80 && upside >= 75)  → "speculative" // 投機的機会
if (certainty < 80 && upside < 75)   → "avoid"       // 回避/様子見
```
