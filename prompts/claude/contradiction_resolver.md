# Claude矛盾解消プロンプト

> **実行条件**: contradictions.jsonに項目がある場合、自動実行

---

## タスク

**Grok/Gemini出力間で検出された矛盾を解消し、正確な情報を確定せよ。**

---

## 矛盾リスト

```json
[contradictions.jsonの内容がここに挿入される]

例:
{
  "contradictions": [
    {
      "id": "C001",
      "topic": "NVDA データセンター売上",
      "grok_value": "$18.4B (Q3 2024)",
      "gemini_value": "$14.5B (Q3 2024)",
      "discrepancy": "$3.9B差異",
      "priority": "high"
    }
  ]
}
```

---

## 解消プロセス

### 各矛盾について:

1. **一次情報源を特定**
   - SEC EDGAR（10-K、10-Q、8-K）
   - 企業プレスリリース
   - 決算説明会資料

2. **正確な値を確定**
   - 一次情報源の値を採用
   - 計算方法/定義の違いを確認

3. **差異の原因を特定**
   - 期間の違い（Q vs FY）
   - 定義の違い（セグメント範囲）
   - 情報の古さ
   - 単純な誤り

---

## 出力形式

```markdown
## 矛盾解消レポート

### [C001]: NVDA データセンター売上

#### 矛盾内容
- Grok: $18.4B (Q3 2024)
- Gemini: $14.5B (Q3 2024)
- 差異: $3.9B

#### 一次情報源確認
- **SEC 10-Q (Q3 FY25)**: $14.51B
- **決算プレスリリース (2024-11-20)**: $14.51B
- **URL**: [SEC EDGAR URL]

#### 確定値
- **正確な値**: $14.51B (Q3 FY25 = 2024年8-10月)
- **確度**: 高（一次情報源確認済）

#### 差異の原因
- Grok: Q4予測値を誤って使用した可能性
- Gemini: 正確

#### 推奨アクション
- consolidated_companies.jsonを$14.51Bに更新
- Grokの該当データに「要確認」フラグ

---

[他の矛盾も同様に処理]
```

---

## 優先順位

| 優先度 | 対象 | 理由 |
|--------|------|------|
| 最高 | Top 30企業関連 | 投資判断に直結 |
| 高 | 市場シェア/独占情報 | 5軸評価に影響 |
| 中 | 財務データ | 資本効率評価に影響 |
| 低 | 一般的な市場データ | 間接的影響 |

---

## 解消できない場合

矛盾を解消できない場合:

1. **両方の値を記録** - 範囲として表現
2. **確度を「低」に設定**
3. **追加調査タスクを生成** - pending.jsonに追加
4. **人間レビューをフラグ** - エスカレーション

```markdown
#### 解消不可
- **理由**: 一次情報源が見つからない
- **暫定値**: $14.5B - $18.4B（範囲）
- **確度**: 低
- **追加タスク**: R-XXX（pending.jsonに追加）
- **エスカレーション**: 要人間レビュー
```
