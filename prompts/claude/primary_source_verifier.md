# Claude一次情報源確認プロンプト

> **実行条件**: 重要情報に一次情報源確認が必要な場合、自動実行

---

## タスク

**重要情報の一次情報源を確認し、確度を向上させよ。**

---

## 確認対象リスト

```json
[確認が必要な項目リストがここに挿入される]

{

    }
```

---

## 一次情報源の定義

| 優先度 | 情報源タイプ | 例 |
|--------|------------|-----|
| A | 政府公式文書 | SEC EDGAR、SAM.gov、Federal Register |
| A | 企業公式発表 | 10-K/Q、プレスリリース、IR資料 |
| B+ | 業界専門メディア | Defense News、SpaceNews（複数で検証） |
| B | 通信社・シンクタンク | Reuters、CSIS、RAND |
| - | ソーシャルメディア | **使用禁止** |

---

## 確認プロセス

### 各項目について:

1. **一次情報源を検索**
   - SEC EDGAR で10-K/10-Q検索
   - 企業IRページ確認
   - 政府契約データベース検索

2. **情報を検証**
   - 数値の正確性
   - 日付/期間の確認
   - 定義/範囲の確認

3. **確度を更新**
   - 一次確認済 → 確度「高」
   - 複数B+ソース一致 → 確度「中」
   - 単一ソースのみ → 確度「低」

## 確認優先順位

| 優先度 | 対象 | 理由 |
|--------|------|------|
| 最高 | 市場シェア | 独占力評価の核心 |
| 最高 | 政府契約 | 政策耐性評価に直結 |
| 高 | 認証/資格 | 参入障壁の根拠 |
| 高 | 売上/利益 | 財務評価に必要 |
| 中 | 成長率予測 | 将来性評価に影響 |

---

## 確認できない場合

一次情報源が見つからない場合:

1. **複数の二次情報源で検証**
2. **確度を「中」または「低」に設定**
3. **情報の限界を明記**
4. **必要に応じて外部LLM調査タスクを生成**

```markdown
#### 一次情報源なし
- **検索結果**: 一次情報源で確認できず
- **代替検証**: [二次情報源1]、[二次情報源2]で一致
- **確度**: 中
- **注記**: 一次情報源での確認が望ましい
- **追加タスク**: Geminiで詳細調査（pending.jsonに追加）
```
