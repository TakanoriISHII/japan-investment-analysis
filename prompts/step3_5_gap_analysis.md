# Step 3.5: Gap分析 & 補足調査（ボトルネック評価対応）

## 目的

4つのLLMから収集した情報を統合した後、**評価前に情報の見落としや不足を検出**し、必要に応じて補足調査を行う。

**核心の問い:**
- この企業の技術がなくなったら、何が止まるか？
- 何社・何業界がこの技術に依存しているか？
- 外国企業が追いつくのに何年かかるか？

---

## チェックリスト

### 1. クロス検証（複数LLMの比較）

```
□ 複数LLMが言及した企業をリストアップ
□ 各LLMの評価・情報量を比較
□ 情報が矛盾または大きく異なる場合 → 補足調査フラグ
```

### 2. ボトルネック情報チェック ★最重要★

新フレームワークの核心である「技術ボトルネック度」の情報が十分か確認:

```
□ 依存企業・業界数が明確か？（何社がこの技術に依存？）
□ 代替技術の有無が調査されているか？
□ 市場シェアが一次資料で確認されているか？
```

**ボトルネック情報が不足している企業を抽出:**
- 「世界シェア○%」のみで、依存企業数が不明
- 代替技術の調査がされていない
- 「業界首位」のみで、具体的な顧客リストがない

**補足調査のクエリ:**
```
「[企業名] [技術名] 顧客」
「[技術名] 代替技術」
「[技術名] なくなったら」
「[企業名] 採用企業 一覧」
```

### 3. 競争優位持続性チェック

```
□ 外国競合リスクが調査されているか？
□ 技術リードタイム（何年で追いつかれるか）が明確か？
□ 地理的・制度的独占性の理由が記載されているか？
```

**不足パターン:**
- 「世界首位」だが、中国・韓国の追い上げ状況が不明
- シェアは高いが、参入障壁の理由（認証、技術蓄積）が不明
- 競合他社の名前すら記載されていない

**補足調査のクエリ:**
```
「[技術名] 中国 韓国 競合」
「[企業名] vs [競合名]」
「[技術名] 新興企業 スタートアップ」
「[技術名] 特許 参入障壁」
```

### 4. 収斂加速性チェック

```
□ 6ドメイン全てについて需要増加の可能性が調査されているか？
□ 「企業が関与している」と「その技術への需要が増える」が区別されているか？
```

**チェック項目（各ドメインについて）:**
```
□ AI投資が増えると、この技術の需要は増えるか？
□ エネルギー投資が増えると、この技術の需要は増えるか？
□ 防衛投資が増えると、この技術の需要は増えるか？
□ ロボティクス投資が増えると、この技術の需要は増えるか？
□ 宇宙投資が増えると、この技術の需要は増えるか？
□ サイバー投資が増えると、この技術の需要は増えるか？
```

**不足パターン:**
- 「AI関連」としか記載されていない（他ドメインへの波及が不明）
- 企業の事業セグメントは記載されているが、技術の需要先が不明

### 5. 高ボトルネック候補の特定

以下の技術キーワードは高ボトルネック度の可能性が高い:

| 技術キーワード | ボトルネック度の高さの理由 | 補足調査のポイント |
|---------------|---------------------------|-------------------|
| **層間絶縁材/基板材料** | 代替困難、全半導体メーカーが依存 | 依存企業リスト、代替技術の有無 |
| **マスク検査/欠陥検査** | 先端プロセスで必須、独占状態 | 競合の開発状況 |
| **精密減速機** | ロボット全てに必要、日本3社寡占 | 中国競合の品質、スイッチングコスト |
| **原子力部品** | 認証10年以上、代替不可能 | 認証の詳細、新規参入の困難さ |
| **HBM材料** | AI需要爆発、限られた供給元 | 増産計画、競合の投資状況 |
| **高純度化学品** | 半導体製造で必須、日本寡占 | 純度要求、製造ノウハウの壁 |

### 6. ランク乖離チェック

```
□ 情報収集時の順位と評価後の順位を比較
□ 情報収集で上位だが評価で圏外の企業を抽出
□ 抽出された企業 → ボトルネック情報不足の可能性
```

**乖離の原因パターン（新フレームワーク）:**
- ボトルネック度の情報不足（依存企業数、代替技術が不明）
- 競争優位持続性の情報不足（外国競合リスクが未調査）
- 収斂加速性の情報不足（1ドメインしか記載されていない）

---

## 補足調査の実行

### 調査対象の優先順位

1. **ボトルネック情報が不足している高シェア企業** → 最優先
2. **外国競合リスクが未調査の企業** → 高優先
3. **収斂加速性が1-2ドメインのみ記載の企業** → 中優先
4. **ランク乖離が大きい企業** → 要確認

### 調査項目（新フレームワーク対応）

各企業について以下を調査:

```markdown
## [企業名]（[証券コード]）

### コア技術
- 技術名: [具体的な技術・製品名]

### 技術ボトルネック度（40点満点）
- **依存企業・業界数**:
  - この技術がなくなったら何社の製品が作れなくなるか？
  - 顧客リスト: [Intel, NVIDIA, AMD, ...]
  - 依存業界: [半導体、自動車、...]
- **代替不可能性**:
  - 代替技術は存在するか？
  - 存在する場合、性能差は？切り替えコストは？
- **市場シェア**: [X%]（一次資料: [ソース]）

### 競争優位持続性（25点満点）
- **技術リードタイム**: 競合が追いつくまで何年？
- **外国競合リスク**:
  - 中国企業: [企業名、開発状況]
  - 韓国企業: [企業名、開発状況]
  - 欧米企業: [企業名、開発状況]
- **地理的・制度的独占性**: なぜ日本でしか作れないか？

### 収斂加速性（20点満点）
- AI投資増 → この技術の需要増？ [Yes/No、理由]
- エネルギー投資増 → [Yes/No、理由]
- 防衛投資増 → [Yes/No、理由]
- ロボティクス投資増 → [Yes/No、理由]
- 宇宙投資増 → [Yes/No、理由]
- サイバー投資増 → [Yes/No、理由]

### 評価への影響
- 従来スコア: [X点]
- 補足調査後の推定スコア: [Y点]
- 変動理由: [...]
```

## 注意事項

1. **「ドメイン関与」と「技術への需要」を区別する**
   - ❌「日立は6ドメインに関与している」
   - ⭕「日立の送配電技術は、AI（DC電力）とエネルギーの成長で需要増」

2. **ボトルネック度の核心は「依存」と「代替不可能性」**
   - 単にシェアが高いだけでは不十分
   - 「この技術がなくなったら何が止まるか」を具体的に

3. **外国競合リスクを必ず調査**
   - シェアが高くても、中国企業が急追中なら減点
   - 逆に、認証障壁で追随困難なら加点

4. **調査は効率的に**
   - 全企業を調査する必要はない
   - チェックリストで抽出された企業のみ
