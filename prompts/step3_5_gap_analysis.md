# Step 3.5: Gap分析 & 補足調査

## 目的

4つのLLMから収集した情報を統合した後、**評価前に情報の見落としや不足を検出**し、必要に応じて補足調査を行う。

これにより、日立や住友電工のような「複数ドメインにまたがる大企業」が情報不足で過小評価されることを防ぐ。

---

## チェックリスト

### 1. クロス検証（複数LLMの比較）

```
□ 複数LLMが言及した企業をリストアップ
□ 各LLMの評価・情報量を比較
□ 情報が矛盾または大きく異なる場合 → 補足調査フラグ
```

**実行方法:**
```
# 2つ以上のLLMが言及した企業を抽出
grep -h "証券コード\|ticker" data/intelligence/raw/company_info/*.md | sort | uniq -c | sort -rn
```

### 2. ドメイン網羅性チェック ★最重要★

```
□ 各企業が6ドメインのうちいくつに関わるか確認
□ 1ドメインのみ記載の大企業（時価総額1兆円以上）を抽出
□ 抽出された企業 → 補足調査対象
```

**対象となりやすい企業の例:**
- 総合電機: 日立製作所、東芝、三菱電機
- 総合重工: 三菱重工、川崎重工、IHI
- 電線・通信: 住友電工、古河電工、フジクラ
- 総合化学: 信越化学、旭化成、三井化学

**補足調査のクエリ:**
```
「[企業名] AI事業」
「[企業名] エネルギー事業」
「[企業名] 防衛事業」
「[企業名] ロボット事業」
「[企業名] 宇宙事業」
「[企業名] サイバーセキュリティ」
```

### 3. シナジー候補の特定

以下の技術キーワードを持つ企業は複数ドメインにまたがる可能性が高い:

| 技術キーワード | 関連ドメイン | 補足調査のポイント |
|---------------|-------------|-------------------|
| 送配電/電力システム | エネルギー + AI（DC電力） | データセンター向け受注状況 |
| 光通信/光部品 | AI + サイバー + 宇宙 | 海底ケーブル、衛星通信への展開 |
| パワー半導体 | エネルギー + ロボティクス + AI | EV、産業機器、サーバー電源 |
| 防衛電子/センサー | 防衛 + サイバー + 宇宙 | 電子戦、ISR、衛星センサー |
| 原子力部品 | エネルギー + 防衛 | SMR、原子力潜水艦 |
| 産業用ロボット | ロボティクス + AI + 防衛 | ヒューマノイド、無人機 |

```
□ 該当する技術キーワードを持つ企業を抽出
□ 1-2ドメインのみ記載されている場合 → 補足調査
```

### 4. ランク乖離チェック

```
□ 情報収集時の順位と評価後の順位を比較
□ 情報収集で上位（1-30位）だが評価で圏外（51位以下）の企業を抽出
□ 抽出された企業 → 情報不足の可能性が高いため補足調査
```

**乖離の原因パターン:**
- シナジー軸の情報不足（ドメイン横断性が記載されていない）
- 独占力の情報不足（シェア、参入障壁が不明）
- 資本効率の情報不足（ROE、FCFが不明）

---

## 補足調査の実行

### 調査対象の優先順位

1. **時価総額1兆円以上で1ドメインのみ記載** → 最優先
2. **複数LLMが言及したが情報が薄い** → 高優先
3. **技術キーワードに該当するがドメイン横断性未記載** → 中優先
4. **ランク乖離が大きい** → 要確認

### 調査項目

各企業について以下を調査:

```markdown
## [企業名]（[証券コード]）

### ドメイン関与度
- AI: [関連製品・サービス、シェア]
- エネルギー: [関連製品・サービス、シェア]
- 防衛: [関連製品・サービス、シェア]
- ロボティクス: [関連製品・サービス、シェア]
- 宇宙: [関連製品・サービス、シェア]
- サイバー: [関連製品・サービス、シェア]

### シナジー評価
- 不可欠なドメイン数: [N]ドメイン
- 収斂による価値増幅: [説明]

### 追加情報
- [調査で発見した重要情報]
```

## 注意事項

1. **全ての大企業を調査する必要はない** — チェックリストに該当する企業のみ
2. **調査は深さより網羅性** — 6ドメインへの関与度を確認することが目的
3. **時間をかけすぎない** — 1企業あたり5-10分が目安
4. **発見は即座に記録** — consolidated_*.json に追記
